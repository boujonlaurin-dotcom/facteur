## Découpage en 3 PRs atomiques

### PR 1 — Backend : pipeline & endpoints

**Titre** : `feat(sources): smart search pipeline with Brave + Mistral fallback`

**Scope** :
- Migration Alembic `source_search_cache` (1 seule head)
- Services de recherche `search/providers/*` (Brave, Reddit, Google News util)
- Orchestrateur `smart_source_search.py` avec pipeline a→f
- Cache Postgres 24h
- 3 nouveaux endpoints sur `sources.py` :
  - `POST /api/sources/smart-search`
  - `GET /api/sources/by-theme/{slug}`
  - `GET /api/sources/themes-followed`
- Config `brave_api_key` + plafonds coût
- Instrumentation metrics + alertes Sentry

**Pas de changement mobile dans cette PR** — les anciens endpoints continuent
de servir l'app actuelle.

**Sortie attendue** : endpoints testables via curl/Postman, couverture tests
unitaires ≥ 80%, CI green.

**Critères de merge** :
- CI green (pytest + lint)
- Peer review APPROVED
- Variable `BRAVE_API_KEY` ajoutée dans Railway (staging + prod)
- Migration SQL exécutée manuellement via Supabase SQL Editor
- Smoke test manuel sur 5 requêtes variées (URL, @handle, r/, texte libre,
  hors catalog)

### Tests E2E / QA manuel

**Via `/validate-feature` + Chrome mobile viewport 390x844** :

Scénarios :
1. **Happy path texte libre** : taper "lenny newsletter" → voir Lenny's
   Newsletter en résultat avec les 3 derniers articles → ajouter → retour au
   feed avec confirmation.
2. **Happy path handle YouTube** : taper "@fireship" → voir Fireship → ajouter.
3. **Happy path subreddit** : taper "r/france" → voir r/france → ajouter.
4. **Happy path URL directe** : coller "https://stratechery.com" → voir
   Stratechery → ajouter.
5. **Désambiguïsation** : taper "stratechery" (sans URL) → voir Stratechery
   en résultat (via Brave ou Mistral).
6. **Empty state thèmes** : ouvrir l'écran sans saisir → voir ses thèmes
   suivis + pépites + exemples.
7. **Drill-down thème** : tap "Tech" → voir Curées + Candidates + Communauté
   (selon disponibilité).
8. **Fallback thèmes vides** : user sans thèmes suivis → voir thèmes par défaut.
9. **Exemple cliquable** : tap "Lenny's newsletter" → champ pré-rempli → submit.
10. **Aucun résultat** : taper "abcdefg123" → message inline + CTA thèmes.
11. **Ajout depuis pépite** : tap sur une pépite → preview → ajouter.
12. **Résilience réseau** : mode avion → message d'erreur explicite + retry.

## Risques & mitigations

| Risque | Probabilité | Impact | Mitigation |
|---|---|---|---|
| **Brave free tier épuisé** avant fin de mois | Faible | Moyen | Plafond à 1800 req/mois, bascule automatique sur (e-f). Upgrade payant si récurrent. |
| **Mistral hallucine des URLs inexistantes** | Moyen | Faible | Validation systématique via `rss_parser.detect_source`. Les URLs invalides sont filtrées serveur. |
| **Google News RSS casse** (endpoint non officiel) | Moyen | Faible | Étape optionnelle ; si échec → couche (f). Monitoring Sentry. |
| **Cache trop agressif** (24h) → résultats obsolètes | Faible | Faible | TTL 24h est un compromis ; possibilité d'invalidation manuelle par admin. |
| **Pollution table `sources`** par candidats IA | Moyen | Moyen | Mistral ne crée **jamais** de row `Source` ; création uniquement au clic "Ajouter" utilisateur, avec `is_curated=False`. |
| **Explosion du coût Mistral** si pipeline mal ordonné | Faible | Élevé | Tests unitaires vérifient que Mistral n'est appelé que quand nécessaire. Plafond mensuel codé en dur. |
| **Régression UX** (champ unique moins clair que tabs) | Moyen | Moyen | Exemples cliquables + placeholder explicite qui mentionne tous les types. Validation QA sur user non technique avant merge PR 2. |
| **Duplication entre "Pépites" et "Communauté"** du thème | Moyen | Faible | Dédup serveur par `source_id` avant envoi ; une source n'apparaît que dans UN groupe. |
| **Latence P95 dégradée** (> 4s) si pipeline complet | Moyen | Moyen | Court-circuits sur catalog + Brave ; timeout 2s par couche avec fallback silencieux ; cache agressif. |
| **Rate limit abusif par user** | Faible | Moyen | 30 req/jour/user + captcha si > seuil (hors scope MVP, v1.1). |

## Suivi post-merge

Après déploiement complet (3 PRs mergées) :

- **Semaine 1** : daily check Sentry + dashboards coût ; top 20 requêtes non
  résolues → backlog curation.
- **Semaine 2** : analyse events analytics → ratio thèmes/pépites/exemples/
  recherche libre ; décision d'ajuster le poids visuel si déséquilibre.
- **Semaine 4** : revue budget réel vs prévisionnel (~0.05 €/mois ciblé) ;
  ajustement des plafonds si nécessaire.
- **Mois 2** : décider de l'opportunité d'une v1.1 (autocomplete live,
  suggestions semantic via embeddings, etc.) selon usage observé.
