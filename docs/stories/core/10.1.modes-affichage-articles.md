# Story 10.1 — Modes d'affichage des articles (Normal / Minimaliste / Ludique)

## Objectif

Permettre à chaque profil d'utilisateur de choisir son confort de lecture :
- **Normal** : rendu actuel.
- **Minimaliste** : texte seul, sans image, plus compact.
- **Ludique** : images mises en avant, textes plus gros.

Réglage dans le profil (pattern thème) + modal de proposition au démarrage
(post-mise à jour pour les existants, 3ᵉ ouverture pour les nouveaux).

## Plan technique

Point de vérité unique : `DisplayModeSpec` (showImages, fontScale, dense,
hauteurs de cartes pour section_fit, titleMaxLinesDelta, thumbSize), dérivé de
`DisplayMode` persisté en Hive (`settings`/`display_mode`), pattern identique à
`ThemeNotifier` (preview/commit/restore).

1. Fondation : `display_mode_provider.dart` + `display_mode_spec.dart` + tests.
2. `section_fit` paramétré par les hauteurs du spec ; branchement dans
   `_capSectionToFit` (flux_continu_provider) via `ref.watch` → fit + snap
   recalculés au changement de mode.
3. Cartes : FluxContinuArticleCard, EssentielHiFiCard, FeedCard (+ call
   sites), FeedCarousel consomment le spec. Interactions (swipe, long-press,
   nudge anchors) intactes par construction (wrappers sur la carte entière).
4. `display_mode_bottom_sheet.dart` + tile « Affichage des articles » dans le
   profil.
5. Modal d'intro : compteur `appOpenCount` (NudgeCounters), flag post-update
   (ChangelogService), nudge `display_mode_intro` (registry + slot
   FirstImpression après notifModal).
6. Calibration QA des hauteurs via logs `[fit-net]` dans les 3 modes.
