# Story 7.7 — Expansion catalogue de sources curées (FR)

## Contexte

Le diag 2026-04-26 (`.context/source-search-diag.md`) a montré que le catalogue prod compte **194 sources actives, 62 curated** seulement. Sur 9 requêtes utilisateur représentatives, 4 ratent complètement le catalog — dont des cibles évidentes (`Hugo Décrypte`, `The Generalist`, `Arrêt sur Images` en curated).

Tant que le catalogue reste mince, la pipeline tombe systématiquement sur les providers externes (Brave / Google News), avec un risque résiduel de listicles ou de réponses vides — même après le filtrage strict introduit dans `bug-source-search-quality.md`.

## Objectif

Porter le catalogue à un seuil critique (~500 sources curées) couvrant :

- Top 50 médias FR (presse écrite, TV, radio, indépendants)
- Top 30 chaînes YouTube info FR
- Top 30 newsletters Substack FR (économie, tech, politique, culture)
- Top 20 podcasts FR
- Subreddits FR pertinents (r/france, r/europe, r/economie, etc.)

## Acceptance criteria

- [ ] Inventaire dans `data/sources_curated_fr.csv` (URL + feed_url + theme + secondary_themes + bias_stance préliminaire)
- [ ] Script `scripts/import_curated_sources.py` (idempotent) ajoute / met à jour les rows en `is_curated = true`
- [ ] Re-run du diag : couverture des 9 requêtes types passe à 9/9 hits curated
- [ ] Mesure post-déploiement via `source_search_logs` : >70 % des recherches court-circuitent au catalog (`layers_called == ['catalog']`)

## À cadrer

- Source de vérité initiale (existant : `scripts/import_sources.py` + `CURATED_FEED_FALLBACKS`).
- Workflow d'enrichissement continu (annotation par PO ? scraper sur top trends ?).
- Politique pour les sources sans RSS officiel (YouTube via handle, Substack via `/feed`).
