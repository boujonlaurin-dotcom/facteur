# 12.1 — Refonte UI & Brise-glace

> Document enfant de [12.1.smart-source-search.story.md](./12.1.smart-source-search.story.md).
> Détaille la refonte de `AddSourceScreen` : champ unique, brise-glace
> thèmes-first, cartes de résultats enrichies.

---

## Principes de design

1. **Un seul champ, zéro friction cognitive** : on retire les 3 onglets
   URL/YouTube/Reddit. L'utilisateur tape ce qu'il a en tête, le serveur fait
   le reste.
2. **Thèmes-first sur l'empty state** : la première chose que l'utilisateur
   voit, ce sont **ses propres thèmes** suivis — chemin le plus court vers
   une source qu'il va vraiment suivre.
3. **Compactage des pépites communautaires** : le carrousel trending devient
   une strip horizontale 1-ligne pour ne pas voler la vedette aux thèmes.
4. **Exemples cliquables** : les exemples textuels remplacent les onglets
   comme indicateurs des types supportés (briser la glace par l'exemple).
5. **Preview obligatoire avant ajout** : cartes riches avec les 3 derniers
   items du feed pour jugement éclairé.

## Interactions & micro-UX

### Champ de recherche

- **Debounce 350ms** : pendant la frappe, pas d'appel serveur jusqu'au silence.
- **Autocomplete léger** (v1.1, pas bloquant MVP) : ILIKE catalog live, 5 résultats
  max, sous le champ. Pas d'appel Brave/Mistral en live.
- **Submit (Enter / bouton)** : déclenche le pipeline complet.
- **Indicateur de chargement** : skeleton des cartes pendant l'appel, non un
  spinner centré (moins frustrant).
- **Clear (✕)** : remet l'empty state.

### Drill-down thème

- Tap thème chip → animation slide-in vers `ThemeSourcesView`.
- Retour flèche ou swipe → re-affiche l'empty state avec même état scroll.
- Dans `ThemeSourcesView`, ajout d'une source → feedback haptique + toast
  confirmation + la source disparaît de la liste (elle est suivie).

### Ajout de source

- Tap "+ Ajouter" sur `SourceResultCard` :
  - Appel `POST /sources/custom` (endpoint existant) si la source n'est pas
    dans le catalog, sinon `POST /sources/{id}/trust`.
  - Transition : carte collapse vers une "tick" animation puis disparaît.
  - Toast "Ajoutée à ton feed".
- Tap "Preview" : expand de la carte avec 10 items (au lieu de 3) et lien
  externe vers la source.

### Accessibilité

- Champ de recherche : `Semantics.label` explicite FR + indication vocalisée
  des types supportés.
- Chips thèmes : `Semantics.button`, tailles tappables ≥ 44x44.
- Cartes : focus order logique (favicon → nom → CTA).

## États d'erreur & edge cases

| Cas | Comportement UI |
|---|---|
| Requête < 2 caractères | Pas d'appel, message inline "Tape au moins 2 caractères" |
| Aucun résultat | "Rien trouvé. Essaie un autre mot-clé ou parcours par thème ↓" + CTA thème explorer |
| Brave 429 / down | Pipeline continue silencieusement, logs serveur, rien côté UI |
| Mistral timeout | Idem |
| Feed discovery échoue sur une URL candidate | La carte n'apparaît pas (filtrée serveur) |
| User hors ligne | "Connecte-toi pour chercher de nouvelles sources" + retry |
| User sans thèmes suivis | Empty state affiche uniquement les pépites et suggestions de thèmes par défaut (ex. "Tech", "Actu FR", "Produit") |

## Animations & transitions

- **Empty state → résultats** : crossfade 200ms, pas de slide.
- **Thème chip → ThemeSourcesView** : Hero animation sur le chip (label
  devient titre d'écran).
- **Ajout de source** : micro-animation "tick" verte + collapse carte sur
  300ms + toast 2s.
- **Preview expand** : height animation 250ms avec `ClipRect`.
