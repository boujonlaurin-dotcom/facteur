# 19.2 — Veille V3 PR2 « Sources quality + custom sources »

## Status
PR ouverte (en review).

## Context

La veille beta V2 (PR #553/#554/#556 mergées) propose des sources via LLM mais
la qualité observée est médiocre :

- côté **médias établis / curés** : pas pertinents par rapport au topic
  individuel — le prompt actuel décourage le mainstream sans pondérer par
  topic (« Évite Le Monde / Le Figaro »), ce qui jette le bébé avec l'eau du
  bain quand un sujet pointu mérite une couverture par un grand quotidien ;
- côté **niches** : OK mais isolés dans une section dédiée, ce qui sur-pondère
  artificiellement leur place et casse l'ergonomie (utilisateur scroll deux
  listes au lieu d'une).

Le PO (Laurin) a en plus demandé que l'utilisateur puisse **ajouter
manuellement une source custom** depuis le Step 3 du flow Veille (sans quitter
le flow). Aujourd'hui, ajouter une source nécessite de quitter le flow,
naviguer vers `AddSourceScreen`, l'ajouter, puis revenir — friction excessive.

PR1 (#561 — critical fixes : row stuck running, 503 propre, mode édition) est
mergée sur `main`. Cette story = PR2.

## Acceptance Criteria

### T7 — Refonte pertinence sources (backend)

1. ✅ `source_suggester.py` produit **une seule liste** rankée (curés + niches
   mélangés). Plus de séparation interne `_followed` / `_niche`.
2. ✅ Le prompt LLM (`_SOURCES_SYSTEM_PROMPT`) :
   - injecte `purpose` + `purpose_other` + `editorial_brief` (déjà fait en V2,
     conservé) ;
   - demande au LLM de pondérer par **topic individuel** (pas seulement par
     thème) ;
   - autorise les médias établis pertinents au topic (suppression de la
     consigne anti-mainstream) ;
   - demande un `relevance_score` (0–1) par source pour le tri.
3. ✅ Post-traitement :
   - dédoublonnage par **domaine racine** (`_root_domain`) — keep highest
     score per domain ;
   - flag `is_already_followed` calculé en joignant les `Source.id`
     retournés avec les `UserSource(user_id=current)` ;
   - tri par `relevance_score` desc.
4. ✅ Schéma API flat :
   `VeilleSourceSuggestionsResponse(sources: list[VeilleSourceSuggestion])` ;
   chaque suggestion porte `is_already_followed: bool` et
   `relevance_score: float | None`.
5. ✅ Tests CI structurels : 10 fixtures dans
   `packages/api/tests/fixtures/veille_eval_cases.json`. Le test paramétré
   `test_veille_source_suggester_eval.py` mocke le LLM avec une réponse canned
   (12 candidats dont 2 doublons de domaine) et vérifie : count > 0, theme
   valide, URLs parseables, pas de doublon par domaine, tri desc,
   `is_already_followed` cohérent. Pas de LLM judge.
6. ✅ Tests existants `test_veille_source_ingestion.py` adaptés au nouveau
   schéma (suppression des assertions `result.followed` / `result.niche`).

### T8 — Sources custom (mobile) + adaptation au schéma flat + nouveau wording

1. ✅ **Refacto reuse** : extraction du widget `SourceAddPanel`
   (`apps/mobile/lib/features/sources/widgets/source_add_panel.dart`) qui
   contient : `SmartSearchField` + filter chips contenu + résultats (cards
   `SourceResultCard` + skeleton) + ouverture de la modale détail.
   Configurable via `padding`, `showIntro`, `showCommunityGems`,
   `showAddedNudge`, `autoFocusSearch`, `onSourceAdded`.
2. ✅ `add_source_screen.dart` est simplifié pour réutiliser `SourceAddPanel`
   (passe de 707 lignes à ~25 lignes — Scaffold + AppBar + panel).
3. ✅ **Step 3 adapté au schéma flat** : liste unique triée par pertinence,
   sous-titre « Classées par pertinence pour ta veille ». Plus de section
   `Tes sources de confiance` / `Sources niches`.
4. ✅ **Badge « CONFIANCE »** appliqué sur chaque card où
   `is_already_followed == true` (remplace `_SuivieStamp`).
5. ✅ **CTA « Connecter / Connectée »** remplace globalement « Suivre /
   Suivie » sur toutes les cards Step 3 (peu importe `is_already_followed`).
   Padding horizontal augmenté (12 → 14) pour absorber les 2 caractères de
   plus de « Connectée ».
6. ✅ **Bouton « Ajouter une source »** en bas de la liste Step 3 (avant la
   sticky CTA « Continuer ») → ouvre `VeilleAddSourceSheet` (un
   `DraggableScrollableSheet` qui héberge `SourceAddPanel` en mode sheet).
   Après ajout réussi : sheet fermé, source ré-injectée dans le state
   Step 3 (`sourcesMeta` + pré-cochée dans `selectedSourceIds`) via
   `addCustomSourceToVeille`.
7. ✅ **Badge RSS** sur `SourcePreviewCard` :
   - `feed_url` non vide → pill verte « RSS détecté » ;
   - `feed_url` vide → pill orange « Pas de flux RSS — articles peuvent
     manquer ».
