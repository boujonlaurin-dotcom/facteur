# Story 17.1 : Carte "L'essentiel du jour" en haut du feed (V0 refonte navigation)

## Status: In Progress

## Story

**As a** utilisateur Facteur ouvrant l'app pour sa tournée du jour,
**I want** trouver mon "Essentiel du jour" directement en tête du flux principal, sans avoir à passer par un onglet dédié,
**so that** ma "tournée du jour" se vit dans une page unique cohérente — l'Essentiel reste accessible via une carte qui pousse une page dédiée, le widget et la notif gèrent leur propre point d'entrée.

## Context

### Problème produit

Aujourd'hui, l'app a 3 onglets bottom nav (Essentiel / Mon flux / Paramètres) qui scindent artificiellement la "tournée du jour" en deux endroits déconnectés (Essentiel ≠ Flux). Le WelcomeTour 3-étapes existe pour expliquer cette séparation. La notif daily ouvre `/digest` (force l'user dans un parcours fermé).

### Solution V0 (1ère étape de l'EPIC "passer de 3 pages à 1")

- **Carte "L'essentiel du jour"** insérée en haut du feed, dans un mini-carousel horizontal préparé pour accueillir d'autres cartes (mode serein, etc.) lors des étapes suivantes de l'EPIC.
- **Tap carte** → push `/digest` en page dédiée (hors ShellRoute, swipe-back natif iOS).
- **Bottom nav** réduit à 2 tabs (Mon flux / Paramètres).
- **WelcomeTour** supprimé entièrement (clean : flag `welcomeTourSeen`, controller, host, slot orchestrateur).
- **DigestWelcomeModal** supprimé (le user atterrit sur `/feed` post-onboarding, pas de modal d'accueil).
- **Notif daily** payload `route:/digest` → `route:/feed` (entrée naturelle = flux).
- **Widget Android** continue de pointer vers `/digest` (continuité — le widget montre les articles du digest).
- **Mode serein** reste en V0 sur les 2 écrans (Digest + Feed), via `SereinToggleChip` existant. Sera repensé dans une étape ultérieure.
- **Nudges premier impact** (`NotificationActivationModal`, `NotificationRenudgeBanner`, `WellInformedPrompt`) déplacés de DigestScreen vers FeedScreen pour rester cohérents avec le nouveau point d'entrée.

### Out-of-scope V0

- Refonte de la barre de filtre.
- Refonte du header du feed (titre "Bonjour Laurin, votre tournée du jour" et date "JEUDI 21 — MAI" — gardés tels quels en V0).
- Modifications du contenu/algorithme du flux.
- Nouveau bouton bas "Prendre du recul" (la closure_screen existante reste).
- Repenser le toggle Mode Serein (on garde `SereinToggleChip` sur les 2 écrans).
- Multi-cartes dans le carousel (V0 = 1 carte, infrastructure prête pour N).

## Acceptance Criteria

1. ✅ Bottom nav affiche **2 tabs** (Mon flux, Paramètres) — l'onglet "Essentiel" est supprimé.
2. ✅ La route `/digest` existe toujours mais est **hors ShellRoute** (pas de bottom nav quand affichée), avec swipe-back natif iOS.
3. ✅ Le feed affiche en tête (entre le sous-titre "Votre flux issu de vos sources de confiance" et la filter bar) une carte "L'essentiel du jour" présentée dans un mini-carousel horizontal (1 carte aujourd'hui).
4. ✅ La carte affiche le titre "L'essentiel du jour", un sous-titre statique `"5 articles · DD MMM"` (date du jour formatée fr_FR), un badge "L'ESSENTIEL", et est tappable.
5. ✅ Tap sur la carte → `context.push(/digest)` ouvre l'écran Digest tel quel (contenu, mode serein, briefing inchangés), avec swipe-back qui ramène au feed (position scroll préservée).
6. ✅ Le redirect post-authentification redirige vers `/feed` (au lieu de `/digest`).
7. ✅ Le redirect post-onboarding redirige vers `/feed` (au lieu de `/digest?first=true`).
8. ✅ Le payload de notif daily ouvre `/feed` (au lieu de `/digest`).
9. ✅ Le widget Android pointe toujours vers `/digest` standalone (deep link `io.supabase.facteur://digest` inchangé).
10. ✅ Le `WelcomeTour` est entièrement supprimé : fichiers `features/welcome_tour/`, flag `welcomeTourSeen` dans `AuthState`, slot `firstImpressionSlot.welcomeTour`. Pas de migration DB requise (clé NudgeService user-scoped reste dormante en SharedPreferences).
11. ✅ Le `DigestWelcomeModal` est entièrement supprimé : fichier widget, nudge `digestWelcome`, query param `?first=true` non plus consommé.
12. ✅ Les nudges premier impact (`NotificationActivationModal`, `NotificationRenudgeBanner`, `WellInformedPrompt`) sont déclenchés sur `/feed` au lieu de `/digest`.
13. ✅ `SereinToggleChip` reste présent sur Digest ET Feed (status quo V0).
14. ✅ Tests verts : `flutter test`, `flutter analyze`. Backend non touché (no-op `pytest`).
