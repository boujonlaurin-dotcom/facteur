# Feature: Epic 11 - Custom Topics Tracking

## 1. Contexte & Valeur

* **Problème :** Les 8 thèmes macro (Tech, Éco, etc.) ne permettent pas de suivre des sujets de niche (ex: "Affaire Epstein", "GPT-5", "Mobilité douce"). L'utilisateur manque de granularité et de contrôle sur son flux.
* **Solution :** Permettre aux utilisateurs de suivre des "Custom Topics" libres, qui influencent leur feed et leur Digest. L'utilisateur doit sentir que c'est "son" algorithme, configurable et transparent.
* **Métriques de succès :** Engagement sur articles matchant un topic suivi ; taux d'adoption du panneau "Mes Intérêts" ; réduction du churn Day 7-30.

---

## 2. Décisions Produit (Issues du Brainstorming)

### 2.1 Taxonomie : "Hybrid Intent" (Approche C)
- L'utilisateur tape un sujet libre (ex: "Guerre en Ukraine").
- Un call LLM one-shot (à la création) génère : **1 slug parent** (parmi les 50 Mistral) + **5-10 keywords** associés + **1 intent description** (pour matching sémantique futur).
- Matching court terme = `slug parent + keywords`. Chemin long terme = embeddings vectoriels.
- Les 50 slugs Mistral restent le squelette de classification interne des articles.

### 2.2 Scoring : "Unified Topic Profile" + Explicit Boost
- **Refonte partielle** : Tous les signaux d'intérêt (temps passé, likes, bookmarks, follows explicites) sont unifiés dans un objet `UserTopicProfile` par topic, avec un score composite unique (-100 à +100).
- Un article matchant un topic suivi reçoit un **Explicit Boost** (multiplicateur ou bonus fixe) dans le scoring existant.
- L'utilisateur peut voir et ajuster ce score via le panneau "Mes Intérêts" (curseurs 4-5 crans).

### 2.3 Discovery : Day 2 + Nudges, pas Onboarding
- **Pas de surcharge au Day 1.** L'onboarding actuel (Thèmes → Sources → GO) reste intact.
- Introduction au **Day 2** via modale + **Nudges post-lecture** (fin d'article / The End screen) : "Vous avez beaucoup lu sur l'IA, en faire un sujet prioritaire ? [Oui] [Non merci]".

### 2.4 Feed UX : Google Discover + Chips de Cluster
- **Topic Headers** (Google Discover style) : Le feed est fragmenté par des headers visuels *"📌 Intelligence Artificielle — 3 articles récents"* suivis de 2-3 cartes liées.
- **Chip de cluster** : Sous une carte représentative, une chip *"▸ 4 autres articles sur l'IA aujourd'hui"* regroupe les articles liés. Règles d'affichage : ≥3 articles du même topic, sources diversifiées, max 2-3 clusters par écran.
- **Double état de la chip** : Si topic suivi → *"📌 IA — 4 autres articles"*. Si non suivi → *"IA — 4 articles [+ Suivre]"*. Un composant, deux états = discovery + subscription.

### 2.5 CTA sur les cartes : Chip Topic remplace "Masquer"
- Le bouton actuel "Masquer" dans le footer de carte est remplacé par une **chip du topic** de l'article (ex: *"IA ☑️"*).
- Cliquer sur cette chip ouvre la page de Personnalisation refactorisée.

### 2.6 Panneau "Mes Intérêts" (Settings refactorisé)
- **Séparation claire** : "Mes Intérêts" (Topics + Sources + Thèmes) est le cœur émotionnel, front & center. "Préférences de lecture" (récence, format, longueur) reste dans Settings, pour les power users, potentiellement hors-scope EPIC 11.
- **Vue groupée** : Les topics et les sources sont affichés ensemble (ex: `IA → [Développer.com 🔥🔥🔥🔥◻️] [The Verge 🔥🔥🔥◻️◻️]`). Curseurs 4-5 crans par élément.
- **CTA en haut de page** : "Configurer son propre algorithme" → Renvoie vers la page complète de tous les intérêts.

### 2.7 Filtres du Feed : Barre Unique Mixte
- Les chips de filtrage en haut du feed mixent **Thèmes macro** ET **Custom Topics** dans une barre unique scrollable.
- Permet de basculer entre "découverte large" (filtre Thème) et "focus précis" (filtre Topic).

### 2.8 Digest : Best-Effort / Out-of-Scope UI
- La **data** est prête : `UserTopicProfile` alimente le scoring du Digest.
- Le **redesign UI** du Digest (intégration Topic Explorer, redirections) est conditionnel à une autre EPIC (résumé d'information). Hors-scope UI pour cette EPIC, in-scope backend.

---

## 3. Scope & Phasing

### Phase 1 : Fondations Data & Discovery (Day 2)
- [ ] Migration DB : `UserTopicProfile` (score composite, slug parent, keywords, source d'intérêt).
- [ ] Endpoint API `/personalization/topics` (GET/POST/PUT/DELETE).
- [ ] Backend : Call LLM one-shot à la création d'un topic (slug parent + keywords).
- [ ] Frontend Settings : Page "Mes Intérêts" refactorisée (vue groupée Topics+Sources, curseurs).
- [ ] Frontend Discovery : Modale Day 2 + Nudges post-lecture.

### Phase 2 : Feed UX (Google Discover + Clusters)
- [ ] Frontend Feed : Topic Headers inter-feed + Chip de cluster "▸ X articles".
- [ ] Frontend Cards : Chip topic (remplace "Masquer") + double état (suivi / non-suivi).
- [ ] Frontend Feed Filters : Barre unique mixte (Thèmes + Custom Topics).

### Phase 3 : Boost Algorithmique
- [ ] Backend Scoring : Explicit Boost basé sur `UserTopicProfile.score`.
- [ ] Backend : Unification des signaux (implicite + explicite) dans `UserTopicProfile`.

### Phase 4 : Digest Backend (Best-Effort)
- [ ] Backend Digest : S'assurer que `UserTopicProfile` alimente le scoring du Digest.
- [ ] Pas de redesign UI Digest (dépend d'une autre EPIC).

---

## 4. Zones à risque

* **Hot News vs Niches :** Comment garantir que l'algo ne noie pas les sujets chauds du jour dans les niches persos, et vice versa ? Le feed doit servir les deux promesses.
* **Match Rate Niches :** Les keywords générés par LLM doivent être de qualité suffisante pour matcher des articles réels. Risque de faux positifs / faux négatifs.
* **Feed Clustering :** Le masquage d'articles regroupés dans une chip doit être cohérent avec les filtres header. Définir le comportement exact.
* **Digest Alignment :** La migration `UserTopicProfile` doit être compatible avec le futur redesign du Digest (autre EPIC).

---

## 5. Questions Ouvertes (Pour Convergence Phase 2)

1. **Hot News vs Niches :** Quel ratio / quelle logique pour alterner sujets chauds et topics persos dans le feed ? (Ex: Le feed affiche toujours les 2-3 articles les plus "urgents" du jour en premier, puis insère les clusters persos ?)
2. **Page Mes Intérêts :** Contenu exact de la page (composants, interactions, data affichée). Nécessite wireframes LQ.
3. **Feed Clustering :** Quand on affiche "> 4 articles sur IA", les articles sont-ils masqués du feed principal ou dupliqués ? Interaction exacte avec les chips de filtrage header ?
