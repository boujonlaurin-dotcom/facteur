# Feature: Epic 11 - Custom Topics Tracking

## 1. Contexte & Valeur (PRD)

* **Problème :** Actuellement limités aux 8 thèmes macro (Tech, Éco, etc.), les utilisateurs manquent de granularité pour suivre des sujets de niche ou spécifiques (ex: IA, Politique climatique) qui les intéressent vraiment.
* **Solution :** Permettre aux utilisateurs de suivre des sujets personnalisés ("Custom Topics") pour influencer leur flux d'actualité et leur Digest quotidien.
* **Métrique de succès :** Hausse du taux d'engagement sur les articles des sujets suivis ; augmentation du nombre d'articles lus par session via la thématique "Vos Sujets" dans le Digest.

## 2. Décisions Produit (Convergence)

1. **Taxonomie "Managed Folksonomy" :** 
   - L'utilisateur s'abonne à des sujets via un système d'auto-complétion riche. Les sujets sont normalisés (liés aux 50+ topics slugs Mistral) pour garantir qu'il y aura du contenu à matcher, évitant le problème du "cold start".
2. **Scoring V1.5 (Multiplier Boost) :** 
   - Plutôt qu'une refonte totale de l'algorithme (trop risquée), nous ajoutons une "Explicit Multiplier Layer". Un article correspondant à un `followed_topic` reçoit un bonus de score substantiel, le propulsant en haut du feed.
3. **Digest Étendu (Expérimentation 10 articles) :** 
   - Le Digest quotidien passe de 5 à 10 articles, structuré en deux blocs visuels : "L'Essentiel" (5 articles Trends/Qualité) suivi de "Vos Sujets" (5 articles de niche poussés par l'Explicit Boost). Cette augmentation du volume est assumée en Phase 1 pour tester l'appétence, avant de reconverger vers un format plus condensé ultérieurement.
4. **Feed Filters & Discovery :**
   - Les Custom Topics deviennent des filtres prioritaires dans le feed. La découverte se fait via des cartes inter-feed ("Voulez-vous suivre X ?") basées sur le comportement (score implicite fort).

## 3. Scope & Phasing

### Phase 1 : Fondations Data & Discovery (Day 2)
- [ ] Migration DB : Ajouter `followed_topics: text[]` dans `user_personalization`.
- [ ] Endpoint API `/personalization/topics` (GET/POST/DELETE) pour gérer la liste.
- [ ] Frontend Settings : Section "Mes Sujets" pour la gestion CRUD.
- [ ] Frontend Discovery : Modale "Day 2" pour introduire la fonctionnalité après l'onboarding initial + Nudges post-lecture d'article (The End screen / Bottom Sheet).

### Phase 2 : UX Feed & Topic Explorer
- [ ] Frontend Feed Filters : Barre de filtres unique mixant les Thèmes macro et les Custom Topics suivis (scrollable horizontalement).
- [ ] Frontend Topic Explorer : Rendre les tags de topics cliquables (dans le Digest ou les Cards) pour ouvrir une vue "Explorer" listant tous les articles récents liés à ce topic précis.

### Phase 3 : Boost Algorithmique
- [ ] Backend Scoring : Injecter l'Explicit Boost dans la pipeline de scoring si l'article contient des topics slugs correspondants.

### Phase 4 : Le Digest Augmenté (10 Articles - Expérimental)
- [ ] Backend Digest Generation : Passer le calcul à 10 articles (5 Essentiels + 5 Custom Topics).
- [ ] Frontend Digest : Gérer la séparation UI "L'Essentiel" vs "Vos Sujets". Assumer l'allongement du temps de lecture dans un premier temps pour mesurer la data d'engagement.

## 4. Zones à risque (Technical Approach)
* **Match Rate :** S'assurer que les slugs injectés dans `followed_topics` correspondent bien au dictionnaire de classification Mistral afin que le boost s'active.
* **Digest Balancing :** Ne pas écraser l'information majeure du jour par des sujets trop "niche". Bien calibrer le ratio (ex: 3 top news / 2 niches).
