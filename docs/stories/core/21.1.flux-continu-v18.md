## Objectif

Refondre la home post-auth en un **scroll continu structuré en 4 sections éditoriales nommées** (Essentiel, Bonnes Nouvelles, Thème perso #1, Thème perso #2), suivi d'une **closing card "FIN DE TOURNÉE"**, puis du **feed standard paginé** existant pour continuité de scroll.

L'écran remplace `FeedScreen` (route `/feed` actuelle) comme home post-auth. Le digest éditorial existant et le feed restent accessibles via leurs routes (cohabitation propre).

## Architecture cible

```
[FluxContinuScreen]  ─ home post-auth, remplace l'usage de FeedScreen
    │
    ├─ 3 calls parallèles au mount (Future.wait) :
    │     • GET /api/digest/both                  → DualDigestResponse {normal, serein}
    │     • GET /api/users/top-themes             → [TopTheme] top 2 par poids descendant
    │     • GET /api/feed?limit=20                → feed standard (premier batch sous closing)
    │
    ├─ Puis 2 calls dépendants en parallèle :
    │     • GET /api/feed?theme={top1}&limit=5    → section #3
    │     • GET /api/feed?theme={top2}&limit=5    → section #4
    │
    ├─ FluxContinuProvider compose FluxContinuState
    │     { sections: List<FluxSection>, feedContinu: List<Content>, isSerene, moreOpen }
    │
    └─ UI (CustomScrollView slivers, Stack pour sticky superposé) :
        • Header FeedScreen (StreakIndicator + FacteurLogo + ProfileAvatarButton)
        • 5 notifs (Renudge, WellInformed, LettresBanner, DigestEntry, KeywordSuggest)
        • Pour chaque section (FluxSection sealed) :
            SectionBanner (gradient teinté + trait + illustration fade + title Fraunces + blurb)
            + N FluxContinuArticleCard (1 par topic ou par item)
            + PlusDeButton (dashed border, expand)
            + SectionHairline (gradient 1px)
        • ClosingCardV18 ("FIN DE TOURNÉE" stamp Courier Prime rotated -2deg)
        • Feed standard items (paginé via FeedRepository, dédupliqué)
        • StickyTabBar superposé via Stack quand scrollOffset > 60
```

## Modèles (sealed class)

```dart
sealed class FluxSection {
  final SectionKind kind;
  final String label;
  final Color accent;
  final String? illustrationAsset;
  final int coreVisibleCount;
  int get totalCount;
  bool get hasOverflow => totalCount > coreVisibleCount;
}

class DigestTopicSection extends FluxSection {
  final List<DigestTopic> topics;  // 1 carte par topic, lead via pickTopicLead
}

class FeedThemeSection extends FluxSection {
  final String? themeSlug;
  final List<Content> items;
}

/// Picks the lead article for a digest topic (followed-source first, else
/// the pivot rank=1). Ported from digest/widgets/topic_section.dart.
DigestItem pickTopicLead(DigestTopic topic);
```

Le `SectionBlock` fait un switch exhaustif sur la sealed class pour produire la liste de cartes.

## Mode serein (swap mécanique #1↔#2)

Le toggle `serein_toggle_provider.dart` reste source de vérité. Le provider FluxContinu écoute le toggle (`ref.listen`) et appelle `invalidateSelf` au changement pour refaire les calls (le `serein` paramètre du `/feed` doit être à jour côté backend). Le swap est appliqué localement dans `_compose(isSerene)`.

## "Plus de…" — expand/collapse local

État local `Map<SectionKind, bool> moreOpen` dans le provider. Tap sur `PlusDeButton` → toggle. `SectionBlock` lit `topics.take(coreVisibleCount)` (ou `items.take(...)`) toujours, puis `topics.skip(coreVisibleCount)` si `moreOpen[kind] == true`.

| Section | coreVisibleCount | Plus de… révèle |
|---|---|---|
| Essentiel | 3 | jusqu'aux topics restants |
| Bonnes Nouvelles | 2 | jusqu'aux topics restants |
| Thème #1, #2 | 2 | jusqu'aux items restants |

Si la section a ≤ coreVisibleCount → `hasOverflow == false`, bouton masqué.

## Feed standard sous closing — dédup

```dart
List<Content> _dedupFeed(List<Content> feed, List<FluxSection> sections) {
  final seen = <String>{};
  for (final section in sections) {
    switch (section) {
      case DigestTopicSection(:final topics):
        for (final topic in topics) {
          if (topic.articles.isEmpty) continue;
          seen.add(pickTopicLead(topic).contentId);
        }
      case FeedThemeSection(:final items):
        for (final item in items) seen.add(item.id);
    }
  }
  return feed.where((c) => !seen.contains(c.id)).toList();
}
```

## Fallbacks et edge cases

| Cas | Comportement |
|---|---|
| `top-themes` retourne `[]` | Sections 3 & 4 = `tech` + `environment` |
| `top-themes` retourne 1 seul thème | Section #4 = fallback complémentaire |
| `/feed?theme=X` retourne < 2 articles | Masquer la section perso (et son tab) |
| `/digest/both.normal` null ou topics vides | Masquer Essentiel + tab |
| `/digest/both.serein` null ou topics vides | Masquer Bonnes + tab |
| Tous les calls échouent | Error state global : "Le flux continu est indisponible. Réessayer" |
| 1-3 calls échouent | Afficher les sections dispo, log Sentry pour les calls KO |
