## Objectif

Refondre la home post-auth en un **scroll continu structuré en 4 sections éditoriales nommées** (Essentiel, Bonnes Nouvelles, Thème perso #1, Thème perso #2), suivi d'une **closing card "FIN DE TOURNÉE"**, puis du **feed standard paginé** existant pour continuité de scroll.

L'écran remplace `FeedScreen` (route `/feed` actuelle) comme home post-auth. Le digest éditorial existant et le feed restent accessibles via leurs routes (cohabitation propre).

## Décisions PM tranchées (2026-05-13)

## Architecture cible

```
[FluxContinuScreen]  ─ remplace FeedScreen comme home post-auth
    │
    ├─ 3 calls parallèles au mount (Future.wait) :
    │     • GET /api/digest/both                  → DualDigestResponse {normal, serein, serein_enabled}
    │     • GET /api/users/top-themes             → [TopThemeResponse] top 2 par poids descendant
    │     • GET /api/feed?limit=20&offset=0       → feed standard (premier batch sous closing)
    │
    ├─ Puis 2 calls dépendants en parallèle :
    │     • GET /api/feed?theme={top1}&limit=5    → section #3
    │     • GET /api/feed?theme={top2}&limit=5    → section #4
    │
    ├─ FluxContinuProvider agrège en FluxContinuState
    │     { essentiel, bonnes, themePerso1, themePerso2, feedContinu, isSerene, moreOpen: {SectionKind: bool} }
    │
    └─ UI (SingleChildScrollView + Stack pour sticky) :
        • FluxAppBar (absolute top:32 — streak + Facteur wordmark + avatar 30×30)
        • Top spacer 76px
        • Pour chaque section (4) :
            SectionBanner (gradient teinté + rule line + illustration + title Fraunces + blurb)
            + 2-3 DigestCards (adaptées V1.8)
            + PlusDeButton (dashed border, expand)
            + SectionHairline (gradient 1px)
        • ClosingCardV18 ("FIN DE TOURNÉE" stamp Courier Prime rotated -2deg)
        • Feed standard items (paginé, dédupliqué par content_id)
        • StickyTabBar (NotificationListener bascule AppBar ↔ StickyBar quand scrollOffset > 60)
```

## Mode serein (swap mécanique #1↔#2)

```dart
// flux_continu_provider.dart
List<Section> _orderedSections(FluxContinuState s) {
  final base = [s.essentiel, s.bonnes, s.theme1, s.theme2]
      .whereType<Section>().toList(); // skip nulls (cf. fallbacks)
  if (s.isSerene && base.length >= 2) {
    return [base[1], base[0], ...base.sublist(2)];
  }
  return base;
}
```

Le toggle `serein_toggle_provider.dart` reste source de vérité. Pas de logique propre au flux.

## "Plus de…" — expand/collapse local

État local Riverpod `Map<SectionKind, bool> moreOpen` dans le provider. Tap sur `PlusDeButton` → toggle. `SectionBlock` lit `articles.take(coreCount)` toujours, puis `articles.skip(coreCount)` si `moreOpen[kind] == true`.

| Section | coreCount | Plus de… révèle |
|---|---|---|
| Essentiel | 3 | jusqu'à 2 articles restants |
| Bonnes Nouvelles | 2 | jusqu'à 3 articles restants |
| Thème #1, #2 | 2 | jusqu'à 3 articles restants |

Si la section a ≤ coreCount articles → masquer le bouton "Plus de…".

## Sticky tab bar V1.8

- `NotificationListener<ScrollNotification>` dans `FluxContinuScreen` → expose `scrollOffset` et `scrollPct` via state local.
- AppBar disparaît, sticky apparaît quand `scrollOffset > 60`.
- 4 tabs scrollables horizontalement. Active = couleur section + underline 2px + dot. Done = gris muet + strike-through.
- Tap = `Scrollable.ensureVisible(sectionKey, alignment: 0.0, duration: 300ms, curve: Curves.easeInOut)` avec offset négatif -86px.
- Progress fill : `CustomPainter` qui dessine un gradient multi-stop fixe (`#D35400 → #C2185B → #2C3E50 → #6C3483`) pleine largeur, puis clip à `scrollPct × width`. Hauteur 4px en bas du sticky bar.
- Détection section active : `GlobalKey` sur chaque `SectionBanner` + `RenderBox.localToGlobal` au scroll throttlé (~100ms). Section dont le banner est le plus proche de offset 100px du top du viewport.

## Closing card V1.8

- Container `--surface` (token), border-radius 16px, margin 30px 18px 24px, padding 28px 22px 22px, shadow soft.
- Stamp "FIN DE TOURNÉE" : Courier Prime 700 10px, color `#2E7D32`, border 1.5px solid currentColor, padding 4px 8px, `Transform.rotate(angle: -2 * pi / 180)`.
- Heading Fraunces 700 24px, centré.
- Texte stats : "Vous êtes à jour. N étapes · {min} min" (N = nombre total d'articles vus dans les 4 sections, min = sum reading_time_seconds / 60).
- 1 CTA primary "Continuer le feed" → scroll vers le feed paginé en dessous.
- Pas de CTA "Refermer" (le feed continue naturellement, l'utilisateur retournera à la home via l'AppBar normalement ou backstack).

## Fallbacks et edge cases

| Cas | Comportement |
|---|---|
| `top-themes` retourne `[]` | Sections 3 & 4 = `tech` + `environment` (themeMap[slug]) |
| `top-themes` retourne 1 seul thème | Section #4 = fallback `environment` (ou `tech` si top1 == environment) |
| `/feed?theme=X` retourne < 2 articles | Masquer la section perso (et son tab) |
| `/digest/both.normal` null | Masquer Essentiel + tab (si même `.serein` null → error state global) |
| `/digest/both.serein` null | Masquer Bonnes + tab |
| Mode serein + section masquée | Le swap respecte la liste effective (cf. `_orderedSections`) |
| Tous les calls échouent | Error state global : "Le flux continu est indisponible. Réessayer" + retry button |
| 1-3 calls échouent | Afficher les sections dispo, log Sentry pour les calls KO (degraded mode silencieux) |
