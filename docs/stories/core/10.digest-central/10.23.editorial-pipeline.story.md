# Story 10.23 — Editorial Digest Pipeline (Curation + Matching)

## Summary

Pipeline de curation éditorialisée : sélection de 3 sujets par LLM + matching d'articles actu (sources user) et deep (sources profondes) pour le "pas de recul".

## Status: IN PROGRESS

## Dependencies

- **Story 10.22** (DONE): `source_tier` field + 19 deep sources in DB

## Technical Approach

### Architecture Global vs Per-User

- **Global** (1x/batch): clustering + LLM curation + deep matching = 4 LLM calls/jour
- **Per-user**: actu matching only (no LLM, pure in-memory)
- **Cost**: ~$0.03/jour total (not per user)

### Pipeline Flow

```
ÉTAPE 1: ImportanceDetector.build_topic_clusters() [reuse existing]
    ↓
ÉTAPE 2: CurationService.select_topics() [LLM → 3 SelectedTopic]
    ↓ (parallel)
ÉTAPE 3A: ActuMatcher.match_for_user() [per-user, no LLM]
ÉTAPE 3B: DeepMatcher.match_for_topics() [global, 3 LLM calls]
    ↓
EditorialPipelineResult → DailyDigest (editorial_v1 format)
```

### LLM Integration

- Anthropic Claude Sonnet via httpx direct (same pattern as Mistral classification)
- Prompts externalized in YAML (config/editorial_prompts.yaml)
- Fallback: deterministic selection if LLM unavailable
