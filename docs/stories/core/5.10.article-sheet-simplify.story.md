# 5.10 — Simplification du drawer ArticleSheet

**Type** : Feature (UX refactoring)
**Epic** : 5 — Reader / Article Detail
**Fichier principal** : `apps/mobile/lib/features/custom_topics/widgets/topic_chip.dart`

---

## Contexte

Le drawer `ArticleSheet` (ouvert en tapant un tag dans l'écran de lecture) contient 5 sections collapsibles : Topic → Source → Entities → Breakdown → Personnaliser. Il est perçu comme trop complexe, trop long à naviguer, et visuellement chargé.

---

## Décisions prises

| Question | Décision |
|---|---|
| Ordre des sections | Fixe : Topic → Source → Entities → Breakdown. Scroll+open vers la section d'entrée via `initialSection`. |
| "Personnaliser mon flux" | Supprimer la section collapsible. Déplacer "J'ai déjà vu cet article" en footer link discret. |
| "Pourquoi cet article ?" | Pas touché pour l'instant. |
| Simplification visuelle | 2 passes : A (chirurgie légère) puis B (refonte complète) dans le même PR. |

---

## Plan technique

### Pass A — Chirurgie légère

- [ ] **Supprimer les suffixes `(sujet)` / `(source)`** dans les `RichText` des en-têtes de section
- [ ] **Transformer "Personnaliser mon flux" en footer link**
  - Supprimer la section 5 (`_personalizeExpanded`, `_buildSectionHeader` + `_buildActionOption`)
  - Supprimer `ArticleSheetSection.personalize` de l'enum et son usage dans `initState`
  - Ajouter en bas du sheet un `TextButton` discret `eyeClosed` "J'ai déjà vu cet article" (même logique que l'action actuelle)
- [ ] **Consolider les footer links "Gérer"**
  - Supprimer les deux `TextButton.icon` "Gérer mes intérêts" / "Gérer mes sources" dans leurs sections respectives
  - Ajouter un seul footer link "Gérer mes sujets et sources →" tout en bas (pointe vers `RouteNames.myInterests`)
- [ ] **Scroll vers la section d'entrée**
  - Ajouter `ScrollController _scrollController` sur le `SingleChildScrollView`
  - Ajouter `GlobalKey` sur chaque section header : `_topicKey`, `_sourceKey`, `_entitiesKey`, `_breakdownKey`
  - Dans `initState`, `addPostFrameCallback` → `Scrollable.ensureVisible(key.currentContext!, duration: 250ms, curve: Curves.easeOut)` sur la clé correspondant à `initialSection`

### Pass B — Refonte visuelle

- [ ] **Supprimer les frames terracotta**
  - Retirer le `Container(decoration: BoxDecoration(color: _terracotta.withOpacity(0.08), border: ...))` qui entoure le slider "Suivi"
  - Remplacer par `Padding` simple — le badge "Suivi" (point + label coloré) reste le signal de statut
- [ ] **Supprimer les `Divider`** entre sections → remplacer par `SizedBox(height: FacteurSpacing.space4)`
- [ ] **Remplacer les chevrons** `caretUp/caretDown` par une variation typographique
  - Section expanded : titre en `fontWeight: w700`
  - Section collapsed : titre en `fontWeight: w500`, couleur `textSecondary`
  - Supprimer l'icône `PhosphorIcons.caretUp/Down` du `_buildSectionHeader`
