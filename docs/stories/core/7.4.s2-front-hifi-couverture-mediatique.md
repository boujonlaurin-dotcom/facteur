# 7.4 Sprint 2 — Refonte hi-fi front « Couverture médiatique »

## Status
En cours.

## Context

Sprint 1 (PR #616) a livré côté back `highlight_spans` sur
`GET /contents/{id}/perspectives` (diff lexical token-par-token). PR #618
(mini, en cours) ajoute `shared_tokens` par perspective et `reference_pivot`
à la racine de la payload pour permettre le rendu **Mode 3 fidèle** au mockup
hi-fi (shared en gris + key en wash bias + verbe-pivot wash gris).

Côté front, le widget `PerspectivesInlineSection`
(`apps/mobile/lib/features/feed/widgets/perspectives_bottom_sheet.dart`)
utilise encore un design pré-hand-off : bias bar 3 segments mergés (`Gauche` /
`Centre` / `Droite`), CTA en bouton plein, pas de bloc référence, pas de diff
lexical. Cette story refond le widget pour matcher le mockup, ajoute 3 widgets
satellites (`CoverageSpectrumBar`, `DiffTitle`, `DivergenceInlineBadge`) et
câble l'animation d'apparition progressive des surlignages.

## Décisions actées

| Sujet | Décision |
|---|---|
| **Refonte du widget** | In-place dans `PerspectivesInlineSection` (même classe). API publique préservée + 2 props ajoutés : `referenceTitle: String` + `referencePivot: TokenSpan?` |
| **Positionnement** | Inchangé — sous le titre, avant le corps (PR #613 a déjà déplacé) |
| **Diff lexical** | Mode 3 fidèle (shared tertiary + highlight wash bias). Mode 2 (fallback transparent) si `sharedTokens == null` — défense en profondeur si back pas encore déployé |
| **Tri variants** | Inchangé — par bias group gauche → centre → droite |
| **Spectrum bar** | Nouveau widget 5 segments distincts (L/CL/C/CR/R), 96×9 px, sans toucher au `BiasSpectrumBar` 3-segments existant (utilisé ailleurs) |
| **Badge polarisation digest** | Nouveau widget rendu en `CustomPaint` (pas de SVG package), ajouté à `FeedCard` via prop optionnel `divergenceLevel`. Câblé seulement depuis `topic_section.dart` (articles topicalisés). Pépite/Coup de cœur/actu_decalee restent silencieux — conforme au hand-off |
| **Valeurs du badge** | Réutilise `'high'/'medium'/'low'` déjà exposés par le back via `DigestTopic.divergenceLevel`. Pas de nouvelle taxonomie : mapping interne au widget → glyphe + couleur |
| **Animation surlignages** | Cascade séquentielle par titre : tokens fade-in dans l'ordre de `start`, 220 ms par token (`Curves.easeOutCubic`), 25 ms d'écart. Toutes les lignes animent en parallèle au moment de l'expand de la carte. **Trigger 1×** (pas re-déclenché sur scroll/setState). Feeling visé : « Facteur analyse en temps réel pour moi les divergences » |
| **CTA Analyse Facteur** | Refonte du `_buildAnalysisCta` en card dashed border, déprioritée. États loading/done/error préservés (`PerspectivesAnalysisZone`) |
| **DigestCard dead code** | Ignoré — c'est `FeedCard` qui est utilisé partout dans le digest. Plan v3 corrigé sur ce point |

## Acceptance Criteria

1. `PerspectivesInlineSection` rendu hi-fi : bandeau hairlines + `CoverageSpectrumBar`
   + "N médias" + caret, bloc référence (border-left ocre, wash pivot gris si non-null),
   8 lignes variantes (border-left bias + diff lexical), CTA Analyse Facteur dashed.
2. `CoverageSpectrumBar` : 5 segments distincts (L/CL/C/CR/R), 96×9 px, gap 1 px,
   flex proportionnel au `bias_distribution` brut avec floor=1 si valeur nulle.
3. `DiffTitle` : rendu Mode 3 fidèle quand `sharedTokens` non-null ; Mode 2
   fallback (transparent → tertiary pour le hors-key) si null. RichText
   reconstruit token-par-token via offsets `{start, end}`.
4. **Animation cascade** : à l'ouverture de la carte dépliée
   (`isExpanded: false → true`), les tokens shared + key apparaissent
   séquentiellement dans l'ordre de `start`, fade-in 220 ms easeOutCubic,
   25 ms d'écart. Animation 1× par expand, pas re-déclenchée sur rebuild.
5. `DivergenceInlineBadge` : glyphe `CustomPaint` 28×12 + label mono uppercase,
   3 niveaux (`high` = brique-marine paires polarisées + label bold ;
   `medium` = 5 dots étalés ; `low` = 3 dots groupés discrets).
6. `FeedCard` accepte `divergenceLevel: String?` ; badge inséré dans la meta-row
   (poussé à droite via Spacer) ; câblé dans `topic_section.dart` à
   `topic.divergenceLevel`.
7. `content_detail_screen.dart` (2 call-sites) transporte `highlightSpans` +
   `sharedTokens` dans le mapping `PerspectiveData → Perspective` et passe
   `referenceTitle` + `referencePivot` au widget.
8. `flutter test` + `flutter analyze` zéro régression.
9. QA Handoff écrit + `/validate-feature` exécuté ; screenshots de l'animation
   capturés ; rapport APPROVED.

## Hors scope
