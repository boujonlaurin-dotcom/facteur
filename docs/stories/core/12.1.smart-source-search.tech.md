## Pipeline de résolution (ordre optimisé coût)

```
Input texte utilisateur
         │
         ▼
┌─────────────────────────────┐
│ 0. Classification locale    │  regex, zero cost
│    URL? @handle? r/sub?     │
│    texte libre?             │
└────────────┬────────────────┘
             │
   ┌─────────┴──────────┐
   │                    │
   ▼                    ▼
┌──────────┐    ┌────────────────────────┐
│ URL-like │    │ Texte libre / handle   │
│ → detect │    │ → pipeline cascadé     │
│   source │    └──────────┬─────────────┘
│ existant │               │
└──────────┘               ▼
                 ┌──────────────────────┐
                 │ (a) Catalog ILIKE    │ gratuit, ~0ms
                 │     is_curated=True  │
                 │     OR is_curated=F  │
                 └──────────┬───────────┘
                            │
                            ▼
                 ┌──────────────────────┐
                 │ (b) YouTube API      │ gratuit, ~200ms
                 │     (si signal YT)   │ si handle ou "youtube"
                 └──────────┬───────────┘
                            │
                            ▼
                 ┌──────────────────────┐
                 │ (c) Reddit JSON      │ gratuit, ~200ms
                 │     (si signal r/)   │ reddit.com/search.json
                 └──────────┬───────────┘
                            │
                            ▼
                 ┌──────────────────────┐
                 │ (d) Brave Search     │ free tier, ~300ms
                 │     ↓                │ 2000 req/mois
                 │     feed discovery   │ → URLs validées
                 │     sur top 5 URLs   │
                 └──────────┬───────────┘
                            │
                 ≥ 3 résultats ? ── OUI ──► STOP → merge + rank
                            │ NON
                            ▼
                 ┌──────────────────────┐
                 │ (e) Google News RSS  │ gratuit, ~400ms
                 │     news.google.com  │ extraction domaines
                 │     /rss/search      │ → feed discovery
                 └──────────┬───────────┘
                            │
                 ≥ 3 résultats ? ── OUI ──► STOP → merge + rank
                            │ NON
                            ▼
                 ┌──────────────────────┐
                 │ (f) Mistral-small    │ payant, ~1-2s
                 │     contexte = (d)+  │ ~200 tok prompt
                 │     (e) URLs         │ → candidats
                 │     chat_json()      │
                 └──────────┬───────────┘
                            │
                            ▼
                  Validation feed discovery
                  sur chaque URL proposée
                            │
                            ▼
                  Merge + dedup (URL canonique)
                  + rank (score composite)
                            │
                            ▼
                     Top 5-8 résultats
```

### Règles de passage entre étapes

- **Court-circuit sur catalog** : si étape (a) renvoie ≥ 3 résultats curés
  (`is_curated=True`), on sert directement sans appeler les APIs externes.
- **Court-circuit sur URL-like** : regex "domain-like" → feed discovery direct
  via `rss_parser.detect_source` existant (comportement actuel préservé).
- **Étape (f) conditionnelle** : Mistral n'est appelé que si `(a) + (b) + (c) +
  (d) + (e)` ont produit < 3 résultats validés. En pratique, ~15-25% des
  requêtes seulement.
- **Cache 24h partagé** : clé = `sha256(normalize(query))`, valeur = JSON du
  pipeline complet. Persistance en Postgres (table `source_search_cache`).

### Score de ranking composite

```
score = 0.40 × confidence_layer      # (a)=1.0, (b/c)=0.9, (d)=0.8, (e)=0.6, (f)=0.5
      + 0.25 × popularity_catalog    # count(user_sources) normalisé, 0 si hors catalog
      + 0.15 × freshness_feed        # max(0, 1 - days_since_last_item / 30)
      + 0.10 × type_match            # 1.0 si type détecté correspond à intention
      + 0.10 × theme_affinity        # 1.0 si theme ∈ UserInterest du user
```

## Endpoints

### `POST /api/sources/smart-search` (NEW)

**Remplace progressivement** `POST /api/sources/detect` (conservé en
compatibilité pour l'app existante pendant une phase de transition).

**Request body** :
```json
{
  "query": "lenny's newsletter",
  "context": {
    "user_themes": ["product", "tech"]  // optionnel, lu depuis JWT
  }
}
```

**Response** :
```json
{
  "query_normalized": "lenny's newsletter",
  "results": [
    {
      "name": "Lenny's Newsletter",
      "type": "article",
      "url": "https://www.lennysnewsletter.com",
      "feed_url": "https://www.lennysnewsletter.com/feed",
      "favicon_url": "https://www.lennysnewsletter.com/favicon.ico",
      "description": "A newsletter for product builders",
      "in_catalog": true,
      "is_curated": true,
      "source_id": "uuid",
      "recent_items": [
        {"title": "...", "published_at": "2026-04-12T..."},
        {"title": "...", "published_at": "2026-04-09T..."},
        {"title": "...", "published_at": "2026-04-05T..."}
      ],
      "score": 0.92,
      "source_layer": "catalog"
    }
  ],
  "cache_hit": false,
  "layers_called": ["catalog", "brave"],
  "latency_ms": 340
}
```

**Side-effects** :
- Cache insert si pas déjà présent (TTL 24h)
- Metrics increment : `smart_search.layers_called[*]`, `smart_search.latency`
- Nouvelles sources "candidates" découvertes par Mistral → **ne sont PAS**
  insérées en base automatiquement (évite pollution catalog). Elles existent
  uniquement dans la réponse et seront créées avec `is_curated=False` seulement
  si l'utilisateur clique "Ajouter".

### `GET /api/sources/by-theme/{slug}` (NEW)

**Request** : `GET /api/sources/by-theme/tech?limit=8`

**Response** :
```json
{
  "theme": "tech",
  "groups": [
    {
      "label": "Curées",
      "sources": [ {...SourcePreview...} ]
    },
    {
      "label": "Candidates",
      "sources": [ {...SourcePreview...} ]
    },
    {
      "label": "Communauté",
      "sources": [ {...SourcePreview...} ]
    }
  ],
  "total_count": 11
}
```

**Logique serveur** :
1. `SELECT * FROM sources WHERE is_active AND is_curated=True AND (theme=:slug OR :slug = ANY(secondary_themes)) ORDER BY popularity DESC LIMIT :limit`
   → groupe "Curées"
2. Si moins de `:limit` résultats :
   `SELECT * FROM sources WHERE is_active AND is_curated=False AND (theme=:slug OR ...) ORDER BY popularity DESC LIMIT (:limit - count)`
   → groupe "Candidates"
3. Si total < 3 :
   agrégation `user_sources ⋈ user_interests` sur le thème, top N par
   `count(user_id)` → groupe "Communauté". Ces sources sont déjà dans la table
   `sources` (puisque `user_sources` y référence), donc on récupère leur preview.

### `GET /api/sources/themes-followed` (NEW, léger)

Raccourci pratique pour l'empty state : retourne la liste des thèmes suivis
par l'utilisateur courant, avec les `Source` counts par thème.

```json
{
  "themes": [
    {"slug": "tech", "label": "Tech", "followed_sources_count": 7},
    {"slug": "product", "label": "Produit", "followed_sources_count": 3}
  ]
}
```

Dérivé de `UserInterest` (ou `UserTopicProgress` selon ce qui est déjà exposé
côté front — à aligner avec l'existant dans `progress.py:27`).

## Coûts estimés (100-200 users)

Hypothèses : 5-10 nouvelles sources/user/mois → ~1500 recherches/mois, dont
~40% sont des cache hits → **900 recherches uniques/mois**.

| Couche | Appels/mois estimés | Coût unitaire | Coût total |
|---|---|---|---|
| (a) Catalog ILIKE | 900 | 0 | 0 € |
| (b) YouTube API | ~200 (signal YT) | 0 (quota gratuit) | 0 € |
| (c) Reddit JSON | ~100 (signal r/) | 0 | 0 € |
| (d) Brave Search | ~600 | 0 (< free tier 2000) | 0 € |
| (e) Google News RSS | ~200 (fallback) | 0 | 0 € |
| (f) Mistral-small | ~150 × ~400 tok | ~0.3 €/1M tok | ~0.02 € |
| **Total** | | | **~0.05 €/mois** |

**Marge budget** : nettement sous les 4 €/mois cibles, laisse énormément de
place à la croissance. Les dépassements viendraient d'un cache mal calibré
ou d'une invalidation trop agressive → l'instrumentation doit détecter cela.

### Gardes-fous coût

- **Rate limit par user** : max 30 recherches smart/jour (anti-abuse).
- **Plafond mensuel Mistral** : si `mistral_calls_month > 2000` → skip étape
  (f), fallback sur (a-e) uniquement, log warning Sentry.
- **Plafond mensuel Brave** : si `brave_calls_month > 1800` → skip étape (d),
  fallback direct sur (e-f).

## Observabilité

### Metrics (Prometheus / structured logs)

- `smart_search.requests_total{layer, status}`
- `smart_search.latency_ms{layer}` (histogram)
- `smart_search.cache_hit_ratio`
- `smart_search.mistral_tokens_used_total`
- `smart_search.brave_calls_total`

### Sentry alertes

- Brave free tier > 80% consommé → warning
- Mistral budget mois > 2€ → warning
- Pipeline latence P95 > 5s pendant 10min → critical

### Dashboards

Vue dédiée Grafana (ou équivalent) avec : ratio cache, répartition couches,
coût cumulé du mois, top 20 requêtes non résolues (pour curation manuelle).
