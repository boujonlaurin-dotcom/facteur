# Story 7.6: Expansion de la Base de Sources Analysées

## Status
Completed

## Story
**As a** système,
**I want** disposer d'une base large de sources "analysées" (profilées),
**so that** la fonctionnalité de comparaison/perspective puisse identifier plus de liens pertinents, même si ces sources ne sont pas dans le catalogue "Curated" de l'utilisateur.

## Context
Pour que la feature "Comparer" (Story 7.3) soit pertinente, elle doit couvrir un large spectre du paysage médiatique (France Info, TF1, PQR, Presse Inter...). Cependant, pour ne pas noyer l'utilisateur dans l'onboarding, ces sources ne doivent pas toutes apparaître dans la sélection initiale ("Catalogue Curated").

## Acceptance Criteria
1. **Architecture "Single Source of Truth"** :
   - Un seul fichier maître : `sources/sources_master.csv`.
   - Colonne `Status` pilotant le cycle de vie : `ARCHIVED` -> `INDEXED` -> `CURATED`.
2. **Logique 3 États** :
   - **INDEXED** : Sources techniquement valides (RSS/Youtube détecté). Utilisées par le moteur de comparaison. Données min : URL, Feed, Bias, Reliability.
   - **CURATED** : Sources "Trusted". Visibles dans le catalogue. Données min : tout INDEXED + Scores FQS complets (Indep, UX) + Rationale rédigé.
3. **Volume** : Ajout d'environ 100 sources mainstream en statut `INDEXED`.
4. **Promotion** : Possibilité de passer une source de `INDEXED` à `CURATED` simplement en changeant le statut et en complétant les scores dans le CSV.
