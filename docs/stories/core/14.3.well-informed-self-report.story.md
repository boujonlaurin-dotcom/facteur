## Objective

Ajouter une **mesure longitudinale self-reported** de la satisfaction informationnelle des utilisateurs Facteur, en complément des signaux comportementaux (digests complétés, articles lus, feedback par article) déjà instrumentés en 14.1/14.2.

**Question posée (inline, dans le scroll du digest)** :
> *De 1 à 10, à quel point vous sentez-vous bien informés en ce moment ?*
>
> 1 = pas / très mal
> 10 = autant que je l'aimerai / avec des informations de très bonne qualité
>
> *Cette réponse nous aidera à mesurer dans quelle mesure Facteur t'aides à mieux t'informer — et à comprendre ce qu'il manquerait pour toujours mieux t'y aider !*

**Pourquoi maintenant** : avoir ce signal PENDANT le Sprint 2 permet de corréler l'évolution du score avec chaque feature shipped (carousels, perspectives, sources funnel…), et de construire une baseline avant les optimisations Sprint 3+.

Distinct de la PR4 PMF (Sean Ellis "how disappointed if product goes away") mentionnée dans 14.2 — complémentaire, pas substitut.

---

## Décisions produit

| Décision | Choix |
|---|---|
| Placement | Inline dans scroll du digest (~15-20%), pas modale bloquante |
| Input | 10 boutons pills 1..10 (pattern NPS classique) |
| Dismissal | Skip autorisé via icon `x` |
| Cooldown après réponse | **14 jours** |
| Cooldown après skip | **5 jours** |
| Persistence | Table dédiée `user_well_informed_ratings` + mirror event analytics + PostHog |

## Acceptance criteria

- Après install + premier scroll sur digest (24h+ après install pour respecter global nudge cooldown), l'utilisateur voit la carte.
- Tap sur un pill 1-10 → carte disparaît, row insérée en DB, event PostHog émis, **ne réapparaît pas avant 14j**.
- Tap sur `x` → carte disparaît, event `prompt_skipped` émis, **réapparaît après 5j**.
- Score hors bornes (1..10) rejeté côté API avec 422.
- Aucune régression sur les tests existants (`flutter test`, `pytest -v`).
