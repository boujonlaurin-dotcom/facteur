# Story 26.1 — « Mon courrier » devient « Progression » : clarté + gamification + mise en avant (PR1)

## Statut
Code + tests terminés — en attente de validation QA (/validate-feature) puis /go

## Contexte

La feature de progression (« Lettres du Facteur », onglet « Mon courrier ») n'est pas comprise
et pas assez gamifiée. PR1 = clarté + gamification + mise en avant, 100 % côté client
(zéro backend, zéro migration). PR2 (hors scope) = nouveaux niveaux/lettres + classement réel.

Décisions PO :
1. Rename « Mon courrier (progression) » → « Progression » (UI only, routes inchangées).
2. Item de niveau = grades de facteur (Apprenti facteur → Maître facteur).
3. Classement des Facteurs = teaser « bientôt » seulement.
4. Section progression en top de l'écran Profil, avatar cohérent avec le header (RingAvatar).
5. Badge numérique de niveau discret sur l'avatar header.
6. Banner feed plus joli + progression x/n.

## Échelle des grades (client only, dérivée de LetterProgressState)

| Lettres complétées | Niveau | Titre |
|---|---|---|
| 0 | 1 | Apprenti facteur |
| 1 | 2 | Facteur de quartier |
| 2 | 3 | Facteur titulaire |
| 3 | 4 | Facteur de tournée |
| 4+ | 5 | Maître facteur |

⚠️ `letter_0` (bienvenue) est `archived` avec `actions: []` → ne compter que les archivées
avec `actions.isNotEmpty`.
