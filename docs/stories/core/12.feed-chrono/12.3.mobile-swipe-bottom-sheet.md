# Story 12.3 : Mobile — Swipe Left → Bottom Sheet Source (Slider + Mute)

**Epic** : 12 — Feed Chronologique
**Type** : Feature Mobile UX
**Effort Estimé** : 2 jours
**Status** : ⏳ À faire

---

## 👥 User Story

**En tant qu'** utilisateur de Facteur,
**Je veux** ajuster rapidement la fréquence d'une source en swipant à gauche,
**Afin que** je reprenne le contrôle sur mon flux sans quitter le feed.

---

## 📋 Acceptance Criteria

### AC1 : Swipe left ouvre bottom sheet
- [ ] Swipe left sur une card ne supprime plus la card immédiatement
- [ ] Swipe left ouvre un bottom sheet modal centré sur la source
- [ ] Le bottom sheet affiche : logo source + nom + slider + bouton mute + lien "Tous les réglages"

### AC2 : Slider directement ajustable dans le sheet
- [ ] Le `PrioritySlider` (3 niveaux : 0.5/1.0/2.0) est directement dans le sheet
- [ ] Tap sur une position du slider met à jour la valeur immédiatement
- [ ] Changement du slider appelle l'API `PUT /api/sources/{id}/weight` (existant)
- [ ] Success : toast "Source mise à jour"

### AC3 : Action "Masquer cette source"
- [ ] Bouton "Masquer cette source" dans le sheet
- [ ] Tap le mute immédiatement
- [ ] Toast : "Source masquée" + undo possible pendant 5s (existant snackbar behavior)
- [ ] La card disparaît du feed

### AC4 : Lien "Tous les réglages"
- [ ] Un lien "Tous les réglages >" navigate vers l'écran "Sources de confiance"
- [ ] Le sheet se ferme
- [ ] Les réglages de la source sont visibles dans l'écran (changements du sheet reflétés)

### AC5 : Ancien behavior supprimé
- [ ] Le `DismissBanner` inline est supprimé (ou déprécié)
- [ ] Les références à `_activeDismissalId` et `_activeDismissalContent` sont nettoyées
- [ ] Le swipe left ne montre plus "Masquer" ou "Moins de [Source]" en gris

### AC6 : Card ne disparaît qu'après mute explicite
- [ ] Swipe left → sheet ouvre, card reste visible dans le feed
- [ ] Fermer le sheet (swipe down, tap outside) → card reste
- [ ] Mute dans le sheet → card disparaît + undo snackbar

---

## 🔧 Approche Technique

### Fichiers à modifier/créer

1. **Nouveau : `apps/mobile/lib/features/feed/widgets/source_adjust_sheet.dart`**
   - Widget `SourceAdjustSheet` (StatefulWidget ou ConsumerWidget)
   - Paramètres : `source` (Source model), `onSliderChanged` (callback), `onMuted` (callback)
   - Build :
     ```
     ├─ SizedBox(height: 100)
     ├─ Row(
     │   ├─ SourceLogo (32x32)
     │   └─ Column(
     │       ├─ SourceName
     │       └─ SourceTheme
     │   )
     ├─ SizedBox(height: 20)
     ├─ Text("Fréquence")
     ├─ PrioritySlider(
     │   initial: source.priority_multiplier,
     │   onChanged: (val) { ... API call ... }
     │ )
     ├─ SizedBox(height: 20)
     ├─ ElevatedButton("Masquer cette source")
     ├─ SizedBox(height: 10)
     └─ TextButton("Tous les réglages >")
     ```

2. **`apps/mobile/lib/features/feed/screens/feed_screen.dart`**
   - Localiser `_handleSwipeDismiss` (ligne ~282)
   - Remplacer l'implémentation :
     ```dart
     _handleSwipeDismiss(Article article, Source source) async {
       final result = await showModalBottomSheet(
         context: context,
         builder: (_) => SourceAdjustSheet(
           source: source,
           onSliderChanged: (val) { /* API call */ },
           onMuted: () { /* remove card */ },
         ),
       );
       // Pas de setState automatique — le sheet gère le feedback
     }
     ```
   - Supprimer les lignes `setState(() { _activeDismissalId = ... })`

3. **`apps/mobile/lib/features/feed/widgets/swipe_to_open_card.dart`**
   - Vérifier que `onSwipeDismiss` callback est déclenché au swipe left
   - Ne pas supprimer la card automatiquement — laisser le parent (feed_screen) gérer
   - Peut avoir besoin d'un paramètre `shouldAutoRemove: false` (à vérifier)

4. **`apps/mobile/lib/features/feed/widgets/dismiss_banner.dart`**
   - Déprécié — pas supprimé (backward compat si utilisé ailleurs), mais non importé dans feed_screen

5. **`apps/mobile/lib/features/sources/repositories/sources_repository.dart`** (si ne pas déjà fait)
   - Vérifier que `updateSourceWeight(sourceId, multiplier)` existe
   - Sinon, créer endpoint `PUT /api/sources/{source_id}/weight`

### Interactions avec providers

- **`userSourcesProvider`** : Fournit la liste des sources avec leur `priority_multiplier` courant
- **`sourcesRepository`** : Gère les appels API
- **Optimistic UI** : Le slider change localement, l'API est appelée en arrière-plan (ex: via riverpod's `FutureProvider`)

## ⚠️ Risques & Mitigation

| Risque | Mitigation |
|--------|-----------|
| `SwipeToOpenCard` supprime la card | Adapter le callback, pas auto-remove |
| Slider API call échoue | Toast error, revenir à la valeur précédente |
| Sheet z-order issue | Vérifier le `showModalBottomSheet` stack |
| Double-tap mute | Ajouter un flag `_isMuting` pour éviter les doublons |
