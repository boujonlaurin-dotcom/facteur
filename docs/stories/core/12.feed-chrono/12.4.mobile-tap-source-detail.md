# Story 12.4 : Mobile — Tap Source → Source Detail Modal avec Slider

**Epic** : 12 — Feed Chronologique
**Type** : Feature Mobile UX
**Effort Estimé** : 1 jour
**Status** : ⏳ À faire

---

## 👥 User Story

**En tant qu'** utilisateur de Facteur,
**Je veux** tapper sur le nom/logo d'une source dans une card pour ajuster sa fréquence,
**Afin que** je n'ai pas besoin de swiper à gauche pour accéder aux réglages rapides.

---

## 📋 Acceptance Criteria

### AC1 : Source name/logo tappable dans les cards
- [ ] Le nom et/ou logo de la source dans la FeedCard est enveloppé dans un `GestureDetector`
- [ ] Tap ouvre le `SourceDetailModal` existant

### AC2 : SourceDetailModal a un slider
- [ ] Le `SourceDetailModal` affiche le `PrioritySlider` (existant)
- [ ] Le slider montre la valeur courante de la source
- [ ] Tap sur le slider met à jour la valeur
- [ ] Changement appelle l'API (`PUT /api/sources/{source_id}/weight`)

### AC3 : Slider placement
- [ ] Le slider s'affiche entre le FQS Score Card et le texte d'explication
- [ ] Visuel cohérent avec le reste de la source_detail_modal

### AC4 : No breaking changes
- [ ] Les autres boutons du modal restent (Add/Remove trust, Mute, Copy RSS)
- [ ] Aucune régression si source detail modal est utilisé elsewhere

---

## 🔧 Approche Technique

### Fichiers à modifier

1. **`apps/mobile/lib/features/sources/widgets/source_detail_modal.dart`**
   - Ajouter paramètre `priorityMultiplier: double`
   - Ajouter callback `onPriorityChanged: (double) -> void`
   - Ajouter le `PrioritySlider` dans le build (entre FQS et explication)
   - Exemple :
     ```dart
     Column(
       children: [
         SourceLogoCard(...),
         SizedBox(height: 12),
         FQSScoreCard(...),
         SizedBox(height: 16),
         // NOUVEAU :
         if (isTrusted) ...[
           Text("Fréquence"),
           PrioritySlider(
             value: priorityMultiplier,
             onChanged: onPriorityChanged,
           ),
           SizedBox(height: 16),
         ],
         ExplanationText(...),
         ActionButtons(...),
       ]
     )
     ```

2. **`apps/mobile/lib/features/feed/widgets/feed_card.dart`** (ou la card où afficher le source name)
   - Envelopper le source name/logo dans un `GestureDetector`
   - Ajouter `onTap: () => _openSourceDetail(source)`
   - Implémenter `_openSourceDetail()` :
     ```dart
     Future<void> _openSourceDetail(Source source) async {
       final priorityMultiplier = /* load from userSourcesProvider */;
       showModalBottomSheet(
         context: context,
         builder: (_) => SourceDetailModal(
           source: source,
           priorityMultiplier: priorityMultiplier,
           onPriorityChanged: (val) { /* API call */ },
         ),
       );
     }
     ```

3. **`apps/mobile/lib/features/sources/repositories/sources_repository.dart`**
   - Vérifier que `updateSourceWeight()` existe (utilisé par story 12.3)
   - Sinon, créer le endpoint

### État & Providers

- Charger `priorityMultiplier` depuis `userSourcesProvider`
- Le changement via slider appelle l'API et met à jour le provider localement (optimistic UI)

## ⚠️ Risques & Mitigation

| Risque | Mitigation |
|--------|-----------|
| Source name pas assez gros pour tap | Augmenter la hitbox avec Padding |
| Chaîner les API calls (slider + autre action) | Utiliser `async/await` ou riverpod FutureProvider |
| Double-tap sur slider | Ajouter debounce |
