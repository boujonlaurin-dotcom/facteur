# Story 12.6 : Mobile — Labels, Textes & Toast Migration

**Epic** : 12 — Feed Chronologique
**Type** : Feature Mobile UX / Content
**Effort Estimé** : 1 jour
**Status** : ⏳ À faire

---

## 👥 User Story

**En tant qu'** utilisateur de Facteur,
**Je veux** comprendre les changements du feed et voir les labels corrects,
**Afin que** je ne sois pas confus par une UX qui change.

---

## 📋 Acceptance Criteria

### AC1 : Toast one-shot migration
- [ ] Au premier app open après la mise à jour, un toast/banner s'affiche une seule fois
- [ ] Message : "Votre flux est maintenant chronologique. Retrouvez l'ancien tri dans 'Pour vous'."
- [ ] Dismissible (tap ou timeout 5s)
- [ ] Sauvegardé dans `SharedPreferences` avec clé `chrono_feed_migration_seen`
- [ ] Au 2ème app open, le toast ne réapparaît pas

### AC2 : Renaming "Explorer" → "Mon flux"
- [ ] Le label du tab/nav "Explorer" change en "Mon flux"
- [ ] Tous les endroits où "Explorer" apparaît sont mis à jour
- [ ] Grep pour "Explorer" : 0 résultats en navigation (peut rester dans les docs/comments)

### AC3 : Chip "Pour vous" description mise à jour
- [ ] La description affichée sous la chip "Pour vous" est : "Sélection personnalisée par notre algorithme transparent"
- [ ] Cette description s'affiche dans le filter bar quand la chip est sélectionnée

### AC4 : Textes "Mes Intérêts" mis à jour
- [ ] Si "Ton algorithme, tes règles. Facteur apprend de tes lectures." existe, le remplacer
- [ ] Nouveau texte : "Vos centres d'intérêt influencent le digest et le mode Pour vous."
- [ ] Sous-titre/context text mis à jour si applicable

### AC5 : Sous-titre feed
- [ ] Si le feed a un sous-titre "Vos news personnalisées du jour", il peut rester OU être changé en "Vos sources, en temps réel"
- [ ] Décision : à trancher lors de l'implémentation (AC flexible)

### AC6 : Texte du mode chrono dans filter bar
- [ ] Le description par défaut (quand aucune chip n'est sélectionnée) peut dire "Chronologique diversifié" ou "Articles récents de vos sources"
- [ ] Cohérent avec l'explication du nouveau mode

## 🔧 Approche Technique

### Fichiers à modifier

1. **Toast migration** : `apps/mobile/lib/features/feed/screens/feed_screen.dart`
   - Dans `initState()` (ou `build()` si ConsumerWidget) :
   ```dart
   Future<void> _showMigrationToastOnce() async {
     final prefs = await SharedPreferences.getInstance();
     final seen = prefs.getBool('chrono_feed_migration_seen') ?? false;
     if (!seen) {
       ScaffoldMessenger.of(context).showSnackBar(
         SnackBar(
           content: Text('Votre flux est maintenant chronologique...'),
           duration: Duration(seconds: 5),
         ),
       );
       await prefs.setBool('chrono_feed_migration_seen', true);
     }
   }
   ```

2. **"Explorer" → "Mon flux"** :
   - `apps/mobile/lib/shared/widgets/navigation/shell_scaffold.dart` (ligne ~126)
   - Chercher `label: 'Explorer'` → `label: 'Mon flux'`
   - Grep dans tout le codebase pour "Explorer" (titre, nav, deep links, etc.)

3. **Chip "Pour vous" description** :
   - `apps/mobile/lib/features/feed/providers/theme_filters_provider.dart`
   - Utiliser la description mise à jour (déjà fait dans story 12.2)

4. **"Mes Intérêts" texte** :
   - `apps/mobile/lib/features/custom_topics/screens/my_interests_screen.dart`
   - Localiser le hero text et le remplacer

5. **Sous-titre feed** (optionnel) :
   - `apps/mobile/lib/features/feed/screens/feed_screen.dart` (ligne ~453)
   - Peut rester ou être changé

## ⚠️ Risques & Mitigation

| Risque | Mitigation |
|--------|-----------|
| Toast bloque l'UI | Utiliser SnackBar (non-blocking) |
| SharedPreferences not initialized | Attendre `SharedPreferences.getInstance()` |
