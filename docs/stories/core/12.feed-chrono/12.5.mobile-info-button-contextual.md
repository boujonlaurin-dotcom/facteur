# Story 12.5 : Mobile — Bouton Info Contextuel (Chrono vs Pour vous)

**Epic** : 12 — Feed Chronologique
**Type** : Feature Mobile UX
**Effort Estimé** : 1 jour
**Status** : ⏳ À faire

---

## 👥 User Story

**En tant qu'** utilisateur de Facteur,
**Je veux** voir des informations pertinentes selon le mode du feed,
**Afin que** le "pourquoi cet article ?" reste utile en mode chrono et en mode scoré.

---

## 📋 Acceptance Criteria

### AC1 : Info button (ℹ️) contextuel en mode chrono
- [ ] Quand aucune chip n'est sélectionnée (mode chrono par défaut), le bouton ℹ️ ouvre le `SourceAdjustSheet` (story 12.3)
- [ ] Affiche "Fréquence de [Source]" + slider + mute
- [ ] Pas de scoring breakdown

### AC2 : Info button en mode "Pour vous"
- [ ] Quand "Pour vous" chip est sélectionnée, le bouton ℹ️ montre le scoring breakdown existant
- [ ] Affiche les facteurs (Pertinence +50pts, Source +30pts, etc.)
- [ ] Aucun changement au comportement existant

### AC3 : Autres modes inchangés
- [ ] Chips de mode (Rester serein, Perspectives, Longs formats) : le bouton ℹ️ reste inchangé
- [ ] Chips de thème : comportement existant

### AC4 : Mode disponible au widget
- [ ] Le `FeedCard` ou le gestionnaire d'info button reçoit le mode courant
- [ ] Peut utiliser `selectedFilter` du `FeedNotifier` pour déterminer le mode

---

## 🔧 Approche Technique

### Fichiers à modifier

1. **`apps/mobile/lib/features/feed/screens/feed_screen.dart`**
   - Passer `selectedFilter` au `FeedCard` ou à un provider accessible au widget info button
   - Si `selectedFilter == 'pour_vous'` : comportement existant
   - Sinon (mode chrono) : ouvrir `SourceAdjustSheet`

2. **`apps/mobile/lib/features/feed/widgets/feed_card.dart`** (ou article_card, etc.)
   - Localiser le bouton ℹ️
   - Ajouter logique conditionnelle au `onTap` :
     ```dart
     _onInfoTapped() {
       if (feedMode == 'pour_vous') {
         // Ancien behavior : show breakdown
         _showScoringBreakdown(article);
       } else {
         // Mode chrono : show source adjust sheet
         _showSourceAdjustSheet(article.source);
       }
     }
     ```

3. **`apps/mobile/lib/features/feed/providers/feed_provider.dart`**
   - Exposer `selectedFilter` pour accès au widget (déjà faisable via `ref.watch(feedProvider).selectedFilter`)
