## 👥 User Story

**En tant qu'** utilisateur de Facteur,
**Je veux** voir un flux chronologique diversifié par défaut,
**Afin que** je découvre des articles de toutes mes sources sans algorithme opaque.

## 📋 Acceptance Criteria

### AC1 : Enum FeedFilterMode mis à jour
- [ ] `FeedFilterMode` a deux nouvelles valeurs : `CHRONOLOGICAL = "chronological"` et `POUR_VOUS = "pour_vous"`
- [ ] Pas de migration Alembic (StrEnum avec `native_enum=False`)

### AC2 : Algo "Ratio Normalisé" implémenté
- [ ] Nouvelle méthode `_apply_chronological_diversification(candidates, source_priority_multipliers, limit, offset)` dans `RecommendationService`
- [ ] La méthode applique le pipeline 4-passes (ratio → quota → sélection → tri)
- [ ] Les sources au quota leur quota sont conservées intégralement
- [ ] Les sources au-delà du quota sont réduites aux N plus récents (recency-based)
- [ ] Tous les articles retenus triés par `published_at DESC`

### AC3 : Routing dans get_feed()
- [ ] `mode is None` ou `mode == CHRONOLOGICAL` → `_apply_chronological_diversification()`
- [ ] `mode == POUR_VOUS` → ancien scoring 4-piliers (backward compat)
- [ ] `mode == RECENT` → chrono pur sans diversification (deprecated, backward compat)
- [ ] Autres modes (INSPIRATION, PERSPECTIVES, DEEP_DIVE) → inchangés

### AC4 : Priority multipliers chargés correctement
- [ ] `source_priority_multipliers` chargé tôt dans le flow (avant branching)
- [ ] Les valeurs (0.5, 1.0, 2.0) appliquées correctement au quota
- [ ] Sources non-suivies ignorées

### AC5 : Pagination cohérente
- [ ] Le quota est calculé pour chaque page (pool de 200 candidats → page de 20)
- [ ] Offset-based pagination ne crée pas de doublons/sauts
- [ ] Vérification : page 1 + page 2 = articles distincts

### AC6 : Mutes toujours actifs
- [ ] Les sources mutées ne survivent pas au pipeline (filtrées au niveau SQL)
- [ ] Les thèmes/topics mutés exclues des candidats
- [ ] Aucune régression sur le système de mute existant

### Pseudo-code de `_apply_chronological_diversification()`

```python
def _apply_chronological_diversification(
    candidates: list[Article],
    source_priority_multipliers: dict[UUID, float],
    limit: int,
    offset: int
) -> list[Article]:
    """
    Applique la diversification par source avec quotas pondérés.
    """
    # PASSE 1 : Grouper par source et calculer fréquences
    by_source = {}
    for article in candidates:
        if article.source_id not in by_source:
            by_source[article.source_id] = []
        by_source[article.source_id].append(article)

    total = len(candidates)

    # PASSE 2 : Calculer quotas avec multiplicateurs user
    quotas = {}
    for source_id, articles_src in by_source.items():
        ratio = len(articles_src) / total
        multiplier = source_priority_multipliers.get(source_id, 1.0)
        quota = max(1, ceil(ratio * limit * multiplier))  # min 1 article par source
        quotas[source_id] = quota

    # PASSE 3 : Sélectionner articles à conserver
    retained = []
    for source_id, articles_src in by_source.items():
        quota = quotas[source_id]
        # Garder les N plus récents (articles_src est déjà par published_at DESC)
        kept = articles_src[:quota]
        retained.extend(kept)

    # PASSE 4 : Tri final chronologique
    retained.sort(key=lambda a: a.published_at, reverse=True)

    # PASSE 5 : Pagination
    return retained[offset : offset + limit]
```

### Considérations d'implémentation

- **Mutes** : Appliqués en amont dans `_get_candidates()` (unchanged)
- **recommendation_reason** : Peut rester `None` en mode chrono (mobile gère l'affichage)
- **Tests** : Créer `test_chronological_diversification()` avec:
  - Source A : 10 articles, multiplier 1.0 → quota ~5
  - Source B : 2 articles, multiplier 2.0 → quota ~2 (max)
  - Vérifier que l'ordre final est chrono et que les sources sont équilibrées

## ⚠️ Risques & Mitigation

| Risque | Mitigation |
|--------|-----------|
| Quota < 1 pour sources rares | `max(1, quota)` pour toute source suivie |
| Pagination incohérente | Tests multi-page, vérifier `offset` logic |
| Source avec multiplier 0.0 (impossible mais...) | Clamp à [0.5, 2.0] au niveau du modèle |
| Candidats insuffisants pour page | Peut retourner < 20 articles si pool < 20 — acceptable |
