# Story 12.4 — Carrousels thématiques dans le Feed

> **Type** : Feature
> **Branche** : `boujonlaurin-dotcom/feed-carousels`
> **Statut** : 🔵 PLAN — En attente GO

---

## Contexte

Le feed chronologique manque d'engagement. On intercale des carrousels thématiques tous les ~5 articles pour surfacer des contenus groupés (overflow, saved, gems), sans casser le flow vertical. 6 types de carrousels priorisés, réutilisant le composant `TopicSection` du Digest, les overflow metadata existantes, et les endpoints existants.

## Plan technique

### Étape 1 — Backend : Schemas + assemblage carrousels

**Fichiers :**
- `packages/api/app/schemas/feed.py` — Ajouter `CarouselItemBadge`, `CarouselInfo`, champ `carousels` dans `FeedResponse`
- `packages/api/app/services/recommendation_service.py` — Ajouter `_build_carousels()` en Phase 4 (après safety net)
- `packages/api/app/routers/feed.py` — Appeler `_build_carousels()` et inclure dans la réponse

**Logique `_build_carousels()` :**
- Appelée après scoring + diversification + regroupement
- Skip si filtre explicite actif
- Budget partagé avec CTAs (consomme des slots `MAX_TOTAL_CTAS`)
- 6 types par ordre de priorité : `new_source` → `hot` → `favorite` → `deep` → `gems` → `saved`
- Min 3 articles par carrousel, 1 instance par type

### Étape 2 — Flutter : Modèle `FeedCarouselData`

**Nouveau fichier :** `apps/mobile/lib/features/feed/models/carousel_model.dart`
- Classes `CarouselItemBadge` et `FeedCarouselData` avec `fromJson()`

### Étape 3 — Flutter : Widget `FeedCarousel`

**Nouveau fichier :** `apps/mobile/lib/features/feed/widgets/feed_carousel.dart`
- Basé sur `TopicSection` : PageView 0.88, dots animés, height estimation
- Header simplifié : `emoji + titre` en `displaySmall`
- Badges contextuels via `CarouselItemBadge` au-dessus de chaque carte
- Support "Suivre +" pour sources curées (carrousel hot)

### Étape 4 — Flutter : Extension `EditorialBadge`

## Décisions

- Structure identique à `TopicSection` (PageView 0.88, dots, badges)
- Badge contextuel = seul indicateur par carte (pas de sous-titre)
- Min 3 articles par carrousel
- Masqués quand filtre actif
- 1 instance par type par session (reset au refresh)
- Carrousels consomment le budget CTA existant
- Fallback sources curées pour "Actu chaude" chez petits users
- "Actu chaude" (pos 5) ET "Aller plus loin" (pos 15) peuvent coexister sur même entité
