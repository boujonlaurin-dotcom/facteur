# Story 12.2 : Mobile — Inversion des Chips (Chrono défaut + "Pour vous" en chip)

**Epic** : 12 — Feed Chronologique
**Type** : Feature Mobile UI
**Effort Estimé** : 2 jours
**Status** : ⏳ À faire

---

## 👥 User Story

**En tant qu'** utilisateur de Facteur,
**Je veux** voir le flux chronologique par défaut (sans sélectionner une chip),
**Afin que** je vois mes sources en ordre de fraîcheur avec transparence.

---

## 📋 Acceptance Criteria

### AC1 : Chip "Derniers articles" disparue
- [ ] La chip "Derniers articles" n'apparaît plus dans la filter bar
- [ ] Aucune référence à `'recent'` ou `recentFilter` dans le code

### AC2 : Chip "Pour vous" en position #2
- [ ] Une nouvelle chip "Pour vous" apparaît en 2ème position (après customTopics)
- [ ] Son label est "Pour vous"
- [ ] Sa description est "Sélection personnalisée par notre algorithme transparent"
- [ ] Tap sur "Pour vous" active le scoring existant (mode=pour_vous)

### AC3 : Feed par défaut sans chip sélectionnée
- [ ] Au démarrage du app ou au reset de filtre, aucune chip n'est pré-sélectionnée
- [ ] Le feed charge avec `mode=null` (ou pas de paramètre `mode`)
- [ ] Backend route `mode=null` vers le nouvel algo chronologique

### AC4 : Chips personnalisées conservées
- [ ] Les chips de mode (Rester serein, Changer de perspective, Longs formats) restent en position #1
- [ ] Les chips de thème (Tech, Science, etc.) restent accessibles
- [ ] Les chips custom topics restent accessibles
- [ ] Ordre final : [Mode filters] → [Pour vous] → [Custom topics] → [Theme filters]

### AC5 : Filter bar UI cohérente
- [ ] Aucune chip sélectionnée visuellement = le feed affiche le texte descriptif du mode chrono
- [ ] Tap sur "Pour vous" la surbrille en couleur primaire
- [ ] Tap de nouveau sur "Pour vous" la désélectionne (retour au chrono)

---

## 🔧 Approche Technique

### Fichiers à modifier

1. **`apps/mobile/lib/features/feed/providers/theme_filters_provider.dart`**
   - Localiser `recentFilter` (ligne ~54-59)
   - Supprimer/commenter cette constante
   - Créer `pourVousFilter` :
     ```dart
     const pourVousFilter = FilterConfig(
       key: 'pour_vous',
       label: 'Pour vous',
       description: 'Sélection personnalisée par notre algorithme transparent',
     );
     ```
   - Retourner `[pourVousFilter, ...themeFilters]` au lieu de `[recentFilter, ...themeFilters]`

2. **`apps/mobile/lib/features/feed/screens/feed_screen.dart`**
   - Localiser le merge des filtres (lignes ~516-560)
   - Remplacer `recentFilter` par `pourVousFilter` dans la liste
   - Adapter le tri : `[...personalized_filters, pourVousFilter, ...custom_topics, ...other_themes]`

3. **`apps/mobile/lib/features/feed/providers/feed_provider.dart`**
   - Localiser `setFilter(String? filter)` ou équivalent
   - Ajouter logique : si `filter == 'pour_vous'` → envoyer `mode=pour_vous` à l'API
   - Si `filter == null` → pas de paramètre `mode` (laisse le backend par défaut = chrono)

4. **`apps/mobile/lib/features/feed/repositories/feed_repository.dart`**
   - La logique de mapping `filter → queryParam` devrait déjà fonctionner (mode=pour_vous sera envoyé tel quel)
   - Vérifier que `mode` est bien un paramètre optionnel dans la requête

### Considérations

- **Description dynamique** : La filter bar affiche une description sous les chips. Vérifier que "Sélection personnalisée..." s'affiche quand aucune chip n'est sélectionnée (mode chrono).
- **State management** : `selectedFilter == null` doit être permis (pas d'assertion).
- **Backward compat** : Si mobile reçoit une réponse avec `mode=recent` (ancien API), gérer gracieusement.
