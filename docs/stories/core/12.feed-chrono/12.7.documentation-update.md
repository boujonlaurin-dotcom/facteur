## 🔧 Approche Technique

### Fichiers à mettre à jour

1. **`docs/prd.md`**
   - Localiser la section "Feed" ou "Algorithm"
   - Remplacer la description actuelle du scoring par défaut par :
     ```markdown
     ### Feed : Chronologique Diversifié (Mode Défaut)

     Le mode par défaut est un **flux chronologique avec rééquilibrage intelligent des sources**.
     Cet algorithme :
     1. Trie les articles par date décroissante
     2. Rééquilibre les sources prolifiques avec un système de quota basé sur la fréquence
     3. Applique les multiplicateurs de fréquence utilisateur (0.5/1.0/2.0)
     4. Exclut les sources/thèmes/topics mutés

     **Algo "Ratio Normalisé"** :
     - Fréquence relative par source = articles_source / total_candidats
     - Quota = ceil(fréquence × page_size × priority_multiplier)
     - Pour sources > quota : garder les N plus récents
     - Tri final : published_at DESC

     ### Feed : "Pour vous" (Mode Scoring - Secondaire)

     Accessible via la chip "Pour vous", cet ancien mode utilise l'algorithme scoring à 4 piliers
     (Pertinence, Source, Fraîcheur, Qualité) avec bruit Gumbel.
     ```

2. **`docs/architecture.md`**
   - Localiser "Recommendation Service"
   - Ajouter/mettre à jour le section :
     ```markdown
     ### Feed Filtering Architecture

     **Routing** dans `RecommendationService.get_feed()` :
     - `mode is None | CHRONOLOGICAL` → `_apply_chronological_diversification()` (défaut)
     - `mode == POUR_VOUS` → Scoring 4-piliers (legacy, backward compat)
     - `mode == RECENT` → Chrono pur (deprecated, backward compat)
     - Autres modes (INSPIRATION, PERSPECTIVES, DEEP_DIVE) → inchangés

     **Chronological Diversification Path** :
     1. Load source priority multipliers (`UserSource.priority_multiplier`)
     2. Group candidates by source_id
     3. Calculate quota per source with normalized frequency ratio
     4. Retain articles from each source up to its quota
     5. Sort all retained articles by published_at DESC
     6. Paginate with offset-based approach
     ```

3. **`docs/front-end-spec.md`**
   - Localiser la section "Feed UI"
   - Mettre à jour :
     ```markdown
     ### Filter Bar (Chips)

     **Chip order** :
     1. Mode filters (Rester serein, Perspectives, Longs formats)
     2. "Pour vous" (scoring, accessible via chip)
     3. Custom topics (user-followed)
     4. Theme filters (Tech, Science, etc.)

     **Default state** : No chip selected → Chronological diversified feed loads

     ### Swipe Left Gesture

     Swipe left on a feed card → Opens a bottom sheet for source adjustment :
     - Source logo + name
     - Priority slider (0.5 / 1.0 / 2.0) to control frequency
     - "Mute source" button
     - Link to "All settings" → Sources screen

     **Note** : Card does NOT auto-dismiss on swipe. Only dismissed if user mutes.

     ### Tap on Source

     Tap on source name/logo → Opens SourceDetailModal with slider for immediate adjustment.
     ```

4. **`docs/data-architecture/`** (si existe)
   - Clarifier que `source_priority_multiplier` semantics change
   - Old: multiplier × score → New: frequency control (quota × multiplier)

5. **Related Stories** (grep for references)
   - `docs/stories/core/4.7.feed-personalization.story.md` : Note que le bottom sheet remplace le inline banner
   - Any other stories mentioning "Derniers articles" or "algorithm" : update references
