# Spec 13.5-13.6 — Format visuel & interactions

> Document de spec complémentaire à [`13.5-13.6.construire-son-flux.md`](./13.5-13.6.construire-son-flux.md). Couvre l'anatomie visuelle de la carte et les interactions utilisateur.

## Wireframe — État par défaut (replié)

```
┌──────────────────────────────────────────────────────────┐
│  Construire ton flux · Cette semaine                     │
│  ─────────────────────────────────────────────────────   │
│                                                           │
│  🟥 Le Monde — rarement ouvert          ●●● → ●○○  ℹ︎ ✕ │
│                                                           │
│  🟩 Reporterre — souvent ouvert         ●○○ → ●●○  ℹ︎ ✕ │
│                                                           │

│                                                           │

│                                                           │
│  ─────────────────────────────────────────────────────   │
│              [  Valider  ]      [ Plus tard ]            │
└──────────────────────────────────────────────────────────┘
```

## Anatomie de la carte

### En-tête

- **Titre** : « Construire ton flux »
- **Sous-titre / timeline** : « Cette semaine » (fenêtre 7 jours, factuelle, commune à toutes les propositions)

### Corps — liste de propositions (N ≥ 3)

Chaque ligne = une proposition. Ordre : propositions triées par `signal_strength` DESC (les plus pertinentes en haut).

**Structure d'une ligne** :

| Zone | Contenu | Exemple |
|---|---|---|
| Icône | Pastille visuelle du type d'entité | 🟥 logo source / 🏷 tag topic |
| Label | Nom de l'entité (source ou topic) | « Le Monde », « IA générative » |
| Justification | Phrase factuelle courte sur l'usage | « rarement ouvert », « vue mais ignorée » |
| Valeur cible | Proposition visuelle **pré-active** | `●●● → ●○○` (slider priorité), `🔕 Masquer`, `🔔 Suivre` |
| Action « ℹ︎ » | Déplie les stats précises sous la ligne | (voir ci-dessous) |
| Action « ✕ » | Dismiss cette proposition (individuelle) | animation de sortie, shift de la liste |

### Pied de carte

- **Bouton primaire** « Valider » — applique toutes les propositions restantes
- **Bouton secondaire** « Plus tard » — snooze la carte entière (24h cooldown)

## Wireframe — État déplié (une ligne expanded via ℹ︎)

```
│  🟥 Le Monde — rarement ouvert          ●●● → ●○○  ▼ ✕ │
│     ┌─────────────────────────────────────────────────┐ │
│     │ 15 articles affichés · 0 lu · 0 sauvegardé      │ │
│     │ Période : 7 derniers jours                      │ │
│     │ Signal : faible engagement                      │ │
│     └─────────────────────────────────────────────────┘ │
```

**Comportement** :
- Tap sur `ℹ︎` → transforme l'icône en `▼` et déplie un panneau sous la ligne
- Tap sur `▼` → replie
- Un seul panneau déplié à la fois (pour garder la carte compacte)
- Les stats affichées proviennent de `signal_context` de l'API (`articles_shown`, `articles_clicked`, `period_days`)

---

## Règles de phrase de justification

Les phrases doivent être **courtes, factuelles, non-jugeantes**. Correspondances proposées :

| Type de proposition | Signal | Phrase |
|---|---|---|
| `source_priority DOWN` | Peu ou pas d'engagement | « rarement ouvert » |
| `source_priority UP` | Engagement supérieur à la moyenne | « souvent ouvert » |
| `mute_entity` | Exposition sans engagement | « vue mais ignorée » |
| `follow_entity` | Engagement fort sur une entité non suivie | « suivie de près » |

À ajuster/finaliser en phase d'implémentation selon retours design + wording PO.

## Valeurs cibles — Représentation visuelle

### Source priority (slider 3 niveaux)

Représentation : `●●● → ●○○` (transition current → proposed).
- `●●●` = priorité haute (3/3)
- `●●○` = priorité moyenne (2/3)
- `●○○` = priorité basse (1/3)

La valeur proposée est **pré-sélectionnée visuellement** (le second slider est « actif »). L'utilisateur peut tap pour ajuster entre les 3 niveaux (ou accepter tel quel via « Valider »).

### Follow / Mute entity (toggle 2 états)

Représentation : icône + label texte (`🔕 Masquer` / `🔔 Suivre`).
État pré-actif = proposition. Tap inverse l'état (et équivaut à un dismiss si l'utilisateur annule sa propre décision).

## Interactions détaillées

### Dismiss individuel (✕ sur une ligne)

1. Animation de sortie (slide-out horizontal ou fade)
2. La ligne est retirée de la carte
3. Backend : enregistré comme `action: "dismiss"` au moment du « Valider » ou du « Plus tard »
4. Si N propositions restantes < seuil minimum (ex : < 1), fermer la carte avec animation

### Dismiss global (« Plus tard »)

1. Toutes les propositions non-dismissées passent en `status: "snoozed"` (non appliquées)
2. Cooldown carte entière 24h (voir `regles-affichage.md`)
3. La carte disparaît du feed

### Valider

1. Collecte les propositions restantes + leur valeur courante (pré-active ou modifiée)
2. POST `/apply-proposals` avec le tableau d'actions
3. Feedback UI : snackbar « Tes préférences sont mises à jour » + animation de disparition
4. Les contenus du feed en dessous se rafraîchissent avec les nouvelles préférences (pull-to-refresh implicite ou invalidation de cache)

### Modification fine (avant Valider)

- Slider tap sur source_priority → change la valeur proposée
- Toggle sur entity → inverse suivre/masquer
- Aucune confirmation intermédiaire : tout est appliqué au « Valider »

---

## Accessibilité

- Chaque ligne doit être focusable au clavier / lecteur d'écran
- Le bouton `ℹ︎` doit annoncer « Détails de la proposition »
- Le bouton `✕` doit annoncer « Ignorer cette proposition »
- Le pattern 2-étapes (phrase → stats) doit être navigable sans dépendre du geste tactile
