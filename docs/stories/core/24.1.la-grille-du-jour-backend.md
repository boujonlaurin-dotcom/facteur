## 4. Logique métier

### 4.1 `compute_tiles` — comptage d'occurrences (corrige le proto)
```python
def compute_tiles(answer: str, guess: str) -> list[str]:
    n = len(answer)
    result = ["absent"] * n
    counts: dict[str, int] = {}
    for ch in answer:
        counts[ch] = counts.get(ch, 0) + 1
    for i in range(n):                       # passe 1 : bien placées
        if guess[i] == answer[i]:
            result[i] = "place"
            counts[guess[i]] -= 1
    for i in range(n):                       # passe 2 : présentes selon stock restant
        if result[i] == "place":
            continue
        if counts.get(guess[i], 0) > 0:
            result[i] = "present"
            counts[guess[i]] -= 1
    return result
```

### 4.2 Validation d'une proposition
- Normalise : trim, MAJ, dé-accentue.
- `len != length` → refus `valide:false`, `raison:"longueur"`, **essai non consommé**.
- mot ∉ dictionnaire → refus, `raison:"hors_dictionnaire"`, essai non consommé.
- valide → append `guesses`, `attempts += 1`, calcule états. `guess == word` → `solved` ; sinon si `attempts == max_attempts` → `failed`. Renseigne `finished_at`.
- Partie déjà finie → **409** (`deja_termine`). Une partie/jour, pas de rejeu.

### 4.3 Streak (dérivé, non persisté)
Jours consécutifs **joués** (≥ 1 partie créée) en remontant depuis le jour courant Paris, calculé sur les `puzzle_date` distincts de l'utilisateur. +1 si J-1 présent ; rupture = reset. **Ne touche pas** `UserStreak`.

### 4.4 Classement « tournée de quartier » (agrégat du jour)
- `joueurs` = nb parties **terminées** du jour.
- `monScore` = `attempts` si `solved`, sinon `"X"`.
- `distribution` = part (%) par nb d'essais `1..max` + `"X"` (somme ~100).
- `percentile` = rang du joueur en % (moins d'essais = mieux ; `failed` au fond).
- `quartier` = podium 3 anonymisé (hash quotidien salé → initiales), `Toi`/`moi:true` au rang du joueur.

## 5. Contrat d'API (⚠️ IDENTIQUE PR 2)

Router `app/routers/grille.py`, monté `prefix="/api/grille"`. Auth requise. **Clés JSON = noms FR exacts** (les champs Pydantic portent directement ces noms — pas d'`alias_generator` — pour garantir le contrat byte-exact).

### `GET /api/grille/today`
Charge/auto-crée la partie, restaure les essais joués (avec états). `mot`/`pourquoi` = `null` tant que `in_progress`.
```jsonc
{ "date":"2026-05-30","dateAffichee":"Vendredi 30 mai","dateCourt":"Ven. 30 mai",
  "numero":"N°143","longueur":6,"essaisMax":6,"premiereLettre":"C","indice":"…","theme":"…",
  "statut":"in_progress",
  "essais":[ {"mot":"PLACER","etats":["absent","present","absent","present","absent","present"]} ],
  "nbEssais":1,"mot":null,"pourquoi":null,"streak":5,"prochainMotDansSec":48000 }
```
- **404** si aucun puzzle seedé pour le jour (jamais 500).

### `POST /api/grille/today/guess` — body `{ "mot":"PLACER" }`
```jsonc
{ "valide":false,"raison":"longueur" }              // | "hors_dictionnaire" — essai non consommé
{ "valide":true,"etats":["place","present","…"],"statut":"in_progress",
  "nbEssais":3,"mot":null,"pourquoi":null }          // mot/pourquoi remplis sur solved/failed
```
- **409** si partie déjà terminée.

### `GET /api/grille/today/leaderboard`
Dispo une fois la partie **terminée** (sinon **409**).
```jsonc
{ "percentile":12,"joueurs":4218,"monScore":3,
  "distribution":[ {"score":1,"pct":4}, …, {"score":"X","pct":3} ],
  "quartier":[ {"initiales":"A·M","score":2,"rang":1},
               {"initiales":"Toi","score":3,"rang":2,"moi":true},
               {"initiales":"L·B","score":3,"rang":3} ],
  "streak":5 }
```
