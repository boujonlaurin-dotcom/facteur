# Story 12.1 : Aligner les subtopics onboarding sur les topic slugs ML

> **Epic**: [[epic-12-taxonomy-alignment.md]]
> **Priorité**: P0
> **Estimation**: 2-3h

---

## Contexte

Les subtopics présentés à l'utilisateur pendant l'onboarding utilisent des slugs inventés (`crypto`, `social-justice`, `housing`...) qui ne correspondent à aucun des 51 `VALID_TOPIC_SLUGS` du pipeline ML. Résultat : les préférences onboarding ne sont pas exploitables par le scoring.

## Objectif

Remplacer tous les slugs de subtopics par des `VALID_TOPIC_SLUGS` réels, pour que les préférences onboarding matchent directement les topics ML des articles.

## Travail

### 1. Remplacer les slugs dans `available_subtopics.dart`

Table de mapping :

| Ancien slug | Nouveau slug ML | Label adapté |
|-------------|----------------|--------------|
| `crypto` | `finance` | Finance & Crypto |
| `social-justice` | `justice` | Justice sociale |
| `housing` | `realestate` | Immobilier & Logement |
| `energy-transition` | `energy` | Énergie & Transition |
| `macro` | `economy` | Économie |
| `elections` | `politics` | Politique |
| `institutions` | `politics` | ⚠️ Doublon avec elections → supprimer ou remplacer par un autre topic |
| `media-critics` | `media` | Médias |
| `fundamental-research` | `science` | Sciences |
| `applied-science` | `science` | ⚠️ Doublon → supprimer ou remplacer par un autre topic |

### 2. Ajouter des subtopics manquants pertinents

Topics ML populaires non proposés à l'onboarding :
- `gaming` (sous tech)
- `sport` (sous sport — cf. Story 12.2)
- `travel` (sous culture)
- `gastronomy` (sous culture)
- `work` (sous society)
- `cinema` (déjà sous culture ✅)
- `immigration` (sous society, si pertinent UX)

### 3. Résoudre les doublons

Quand deux anciens slugs mappent vers le même topic ML :
- `elections` + `institutions` → `politics` : garder un seul, remplacer l'autre par ex. `europe` ou `geopolitics`
- `fundamental-research` + `applied-science` → `science` : garder un seul, remplacer l'autre par ex. `space` ou `health`

### 4. Valider côté backend

Vérifier dans `packages/api/app/routers/users.py` que le backend accepte ou ignore les subtopics (actuellement stockés tels quels dans onboarding_answers).

## Fichiers à modifier

- `apps/mobile/lib/features/onboarding/data/available_subtopics.dart` — remplacements slugs
- `packages/api/app/routers/users.py` — validation éventuelle
