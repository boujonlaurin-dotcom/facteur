# Story 12.2 : Ajouter le thème "Sport" à l'onboarding

> **Epic**: [[epic-12-taxonomy-alignment.md]]
> **Priorité**: P1
> **Estimation**: 1h

---

## Contexte

Le backend définit 9 thèmes valides (`VALID_THEMES` dans `topic_theme_mapper.py`), dont `sport`. Mais l'onboarding mobile ne propose que 8 thèmes — `sport` est absent. Les utilisateurs intéressés par le sport ne peuvent pas exprimer cette préférence.

## Objectif

Ajouter `sport` comme 9e thème dans l'onboarding, avec ses subtopics et sources associés.

## Travail

### 1. Ajouter le thème dans `AvailableThemes.all`

```dart
const ThemeOption(
  slug: 'sport',
  label: 'Sport',
  emoji: '⚽',
  color: Colors.orange,
),
```

Position suggérée : après `economy` (dernier dans la liste actuelle) ou entre `society` et `environment` pour un ordre logique.

### 2. Ajouter les subtopics sport dans `available_subtopics.dart`

```dart
'sport': [
  const SubtopicOption(slug: 'sport', label: 'Sport', emoji: '⚽'),
],
```

Note : le topic ML `sport` couvre tout le sport (football, JO, tennis, rugby, etc.). Un seul subtopic suffit. Si l'UX demande plus de granularité, on peut ajouter des labels descriptifs mais garder le même slug `sport`.

### 3. Ajouter les sources sport dans `ThemeToSourcesMapping`

```dart
'sport': [
  // À compléter avec les sources sport CURATED disponibles dans la base
  // Ex: "L'Équipe", "RMC Sport", etc. si disponibles
],
```

Vérifier dans `sources_master.csv` ou la base quelles sources sport sont disponibles et CURATED.

## Fichiers à modifier

- `apps/mobile/lib/features/onboarding/providers/onboarding_provider.dart` — `AvailableThemes.all`
- `apps/mobile/lib/features/onboarding/data/available_subtopics.dart` — subtopics sport
- `apps/mobile/lib/features/onboarding/data/theme_to_sources_mapping.dart` — sources sport
