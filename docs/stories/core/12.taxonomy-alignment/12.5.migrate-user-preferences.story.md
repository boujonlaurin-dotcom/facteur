# Story 12.5 : Migrer les données utilisateur existantes

> **Epic**: [[epic-12-taxonomy-alignment.md]]
> **Priorité**: P0
> **Estimation**: 3-4h
> **Dépendances**: Stories 12.1-12.4 déployées en premier

---

## Contexte

Les utilisateurs existants ont des préférences (onboarding subtopics, muted topics/themes) enregistrées avec les **anciens slugs inventés**. Après le déploiement des stories 12.1-12.4, il faut migrer ces données pour qu'elles matchent les nouveaux `VALID_TOPIC_SLUGS`.

## Contrainte clé

**Minimiser les pertes de préférences utilisateur.** Quelques pertes sont acceptables dans le contexte de la refonte du système de recommandations, mais chaque perte doit être justifiée et loggée.

## Travail

### 1. Table de mapping explicite

```python
SLUG_MIGRATION_MAP: dict[str, str] = {
    # Onboarding subtopics
    "crypto": "finance",
    "social-justice": "justice",
    "housing": "realestate",
    "energy-transition": "energy",
    "macro": "economy",
    "elections": "politics",
    "institutions": "politics",  # doublon accepté, dédupliqué après
    "media-critics": "media",
    "fundamental-research": "science",
    "applied-science": "science",  # doublon accepté, dédupliqué après
    # ThemeToSourcesMapping subtopics (si stockés en DB)
    "middle-east": "middleeast",
    "physics": "science",
    "biology": "science",
    "arts": "art",
    "french-politics": "politics",
    "labor": "work",
    "macroeconomics": "economy",
}
```

### 2. Migration Alembic

Nouvelle migration qui :
1. Lit les `onboarding_answers` JSON de chaque utilisateur
2. Remplace les slugs dans `subtopics[]` selon `SLUG_MIGRATION_MAP`
3. Déduplique (si `elections` et `institutions` → 2x `politics`, garder 1 seul)
4. Met à jour `UserPersonalization.muted_topics` avec la même logique
5. Met à jour `UserPersonalization.muted_themes` si nécessaire (a priori pas d'impact car les theme slugs n'ont pas changé)
6. Logge les migrations effectuées et les pertes (slugs non mappés)

### 3. Script dry-run

Avant exécution réelle :
```bash
# Mode dry-run : affiche les changements sans les appliquer
python -m scripts.migrate_taxonomy --dry-run
```

Output attendu :
```
User 123: subtopics ['crypto', 'ai'] → ['finance', 'ai'] (1 migrated, 0 lost)
User 456: subtopics ['housing', 'elections'] → ['realestate', 'politics'] (2 migrated, 0 lost)
User 789: muted_topics ['unknown-slug'] → [] (0 migrated, 1 lost ⚠️)
---
Total: 42 users migrated, 3 slugs lost
```

### 4. Rollback

La migration doit être réversible :
- Stocker l'ancien JSON dans un champ temporaire ou en log avant remplacement
- Ou utiliser le downgrade Alembic
