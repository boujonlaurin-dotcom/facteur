### Signal Computation (Story 13.1)

Fenetre glissante de 7 jours. Pour chaque user :

**Par source :**
- Compter impressions (articles avec last_impressed_at dans la fenetre)
- Compter interactions (status IN (SEEN, CONSUMED) OR is_saved OR reading_progress > 80)
- Ratio = interactions / impressions
- Comparer au priority_multiplier actuel de UserSource

**Par entite :**
- Compter articles contenant l'entite (via Content.entities array overlap)
- Compter interactions sur ces articles
- Si 5+ articles avec 0 interaction -> signal mute
- Si 5+ articles avec interaction -> signal follow

### Proposals Generation (Story 13.2)

A partir des signaux, generer des propositions :

| Type | Condition | Proposed Value |
|------|-----------|----------------|
| source_priority DOWN | ratio < 0.1 AND multiplier >= 1.0 | multiplier * 0.5 (min 0.5) |
| source_priority UP | ratio > 0.5 AND multiplier <= 1.0 | multiplier * 2.0 (max 2.0) |
| mute_entity | 5+ articles, 0 interaction | "mute" |
| follow_entity | 5+ articles, 3+ interactions | "follow" |

- Max 4 propositions par batch
- Diversifier les types (pas 4x source_priority)
- Signal strength = abs(delta) normalisé [0,1]
- Expire les anciennes propositions pending (> 7j)
- Ne pas regenerer si propositions pending existent deja pour meme entite
