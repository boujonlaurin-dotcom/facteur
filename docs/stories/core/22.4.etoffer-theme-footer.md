# Story 22.4 — Footer « Étoffer [thème] » : la section thématique comme vecteur de croissance des sources

> **Type :** Feature · **Statut :** En review · **Cible :** `main` (staging continu)
> **Branche :** `boujonlaurin-dotcom/post-onboarding-quality-ramp`

## Contexte

Montée en qualité **post-onboarding** : faire en sorte que les **sections
thématiques** de la Tournée du matin apportent chaque jour de la news fraîche et
pertinente. Le goulot nommé par le PO : « pas assez de sources par thématique »
→ pool limité pour les recommandations. Le bon véhicule n'est pas le héros
« L'Essentiel » (5 articles transversaux) mais la **section thématique**, là où
l'utilisateur lit déjà.

L'exploration a recadré le périmètre :
- La **boucle de confirmation d'intérêt** (« Choisie pour vous » →
  `promoteSuggestion`/`dismissSuggestion`) est **déjà construite**.
- Le « sauvetage de thème maigre » existe déjà mais ne rend qu'un **CTA
  générique** (`_FavoriteEmptyState`, `section_block.dart`).

⇒ Le vrai gap à fort levier : transformer ce CTA générique en un **vrai vecteur
contextuel de sources de qualité par thème**. C'est l'objet de cette story
(PR n°1 du plan).

## Décisions PO

1. **Vecteur unique** : un footer au pied de section thématique proposant des
   sources non suivies couvrant ce thème, **avec une UI graduée selon notre
   niveau de contrôle éditorial**, + une entrée de recherche.
2. **Hybride avec gates de contrôle clairs.** Facteur ne doit **jamais**
   apparaître comme garant d'une source extrémiste / non vérifiée. Gradation :
   Pépites éditoriales → catalogue évalué → recherche user (non poussée).

## La gradation de contrôle (cœur de la story)

| Tier | Critère (données existantes) | Cadre UI | Branding |
|------|------------------------------|----------|----------|
| **1 — Pépite Facteur** (`facteur_pick`) | `is_pepite_recommendation` + `pepite_for_themes` ⊇ slug | « Recommandé par Facteur » + raison | Fort, assumé |
| **2 — Catalogue évalué** (`quality_catalog`) | `is_curated` ET `reliability ∈ {high, medium}` ET `bias ∉ {alternative, unknown}` | « Source de qualité sur [thème] » + **badge éval visible** (biais + fiabilité) | Neutre, transparent |
| **3 — Recherche** | tout le reste | **jamais poussé** ; accessible seulement via « Chercher une source » | Aucun |

**Gate de sécurité (serveur)** : les tiers poussés (1 & 2) excluent
`reliability ∈ {low, unknown}` et `bias = alternative`. Seuils = **knobs PO**
(`app/services/source_recommendation_gate.py`). Aucune migration DB (tous les
champs existent).

## Implémentation

### Backend
- `app/services/source_recommendation_gate.py` (nouveau) — knobs PO :
  `passes_safety_gate()` (tiers poussés) + `is_quality_catalog()` (Tier 2).
- `app/schemas/source.py` — `RecommendationTier`, `ThemeSuggestionItem`,
  `ThemeSuggestionsResponse`.
- `PepiteService.get_theme_pepite_sources()` — requête **pure** (pas de
  rate-limit ni d'écriture) des pépites d'un thème.
- `GET /sources/suggest-for-theme/{slug}` — Tier 1 (pépites) puis Tier 2
  (catalogue gaté), cap 3, hors sources suivies (`is_trusted`).

### Mobile
- `theme_suggestions_model.dart` — `ThemeSuggestions` / `ThemeSuggestion` /
  `ThemeSuggestionTier`.
- `SourcesRepository.suggestSourcesForTheme()` + `etofferThemeProvider(slug)`
  (family cachée).
- `EtofferThemeFooter` (`flux_continu/widgets/etoffer_theme_footer.dart`) —
  mini-fiches sources avec **badge de tier**, **one-tap follow** (`setSourceState`
  → `followed`), entrée « Chercher une source [thème] » (Tier 3 → `onAddSources`).
  Calqué sur `_FavoriteEmptyState` (carte beige `0xFFE6E1D6`).
- Injection dans `section_block.dart` → `case FeedThemeSection` (gardé par
  `themeSlug != null`, sujets custom conservent le CTA générique) :
  - **thème vide** → footer **déplié** + accroche « Rien de neuf… » ;
  - **thème maigre** (`underfilled`) → footer **déplié** ;
  - **thème riche** → footer **replié** (bouton « Étoffer [thème] », charge à la
    demande, pour ne pas alourdir).

### Effet boucle
La recherche Tier 3 capture des sources non anticipées → sources custom →
community gems → candidates au **run d'éval** (Track B du plan, gate PO) qui
grossit le pool Tier 2. La demande alimente l'offre.

## Ajustement UX post-livraison (2026-06-26) — footer allégé + raccourci catalogue

> Suite au retour PO : l'UX du footer était trop chargée/différenciée. Les CTA
> texte deviennent discrets/power-user, et le bouton de recherche raccourcit le
> parcours en ouvrant directement le **catalogue de sources filtré sur le thème**.
> Les mini-fiches de suggestions Tier 1/2 (badges + « Suivre ») sont inchangées.

- **Cas A — thème riche (replié)** : le bouton « Étoffer X » devient « Ajouter
  plus de sources (X) » et, au tap, ouvre directement le catalogue filtré
  (`onSearch`) — plus de dépli in-place des suggestions ni de fetch
  `etofferThemeProvider`.
- **Cas C — recommandations vides** : suppression de la phrase « Pas encore de
  source recommandée sur X… » → il ne reste que le lien discret « Chercher une
  source X » (`SizedBox.shrink()` à la place du `Text`).
- **Caret « > » du hero** (`section_banner.dart`) : taille 16 → 18 (`bold`
  conservé) pour épaissir le trait et l'aligner sur le poids du titre Fraunces.
- **Destination recherche / « Chercher une source X »** : page d'ajout scrollée
  jusqu'au `CatalogSourcesStrip` déjà **déplié + filtré** sur le thème. Câblage :
  - `CatalogSourcesStrip` : nouveaux params `initialTheme` + `initiallyExpanded`.
  - `SourceAddPanel` : nouveau param `initialCatalogTheme` + `ScrollController` +
    `GlobalKey` → `Scrollable.ensureVisible` post-frame sur l'empty-state.
  - `AddSourceScreen` + route `add` : query param `theme` → `initialCatalogTheme`.
  - `flux_continu_screen.dart` `onAddSources` : pousse le slug en query param,
    gardé par `_isCatalogTheme` (slug ∈ `sourceThemeFilters`) — un slug custom
    ouvre le catalogue non filtré plutôt que vide.
- Tests MAJ : `etoffer_theme_footer_test` (Cas A tap → `onSearch` ; Cas C plus de
  phrase ; label renommé), `section_block_test` (label renommé), nouveau
  `catalog_sources_strip_test` (déplié + filtré). `flutter analyze` clean,
  24 tests verts. Backend inchangé (aucune migration).
