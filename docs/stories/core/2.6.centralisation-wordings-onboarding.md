# Story 2.6: Centralisation des wordings Onboarding

**Status:** Approved

## Story
**As a** développeur / éditeur,
**I want** centraliser tous les textes de l'onboarding dans un seul fichier,
**so that** je puisse modifier les wordings très facilement sans toucher à la logique UI.

## Acceptance Criteria
1. Un fichier `lib/features/onboarding/onboarding_strings.dart` est créé.
2. Tous les titres, sous-titres, libellés d'options et messages de réaction de l'onboarding y sont déplacés.
3. Les widgets des écrans d'onboarding utilisent ces constantes au lieu de chaînes de caractères en dur.
4. Aucun changement fonctionnel n'est perçu par l'utilisateur final.
