# Story 28.1 — Rituel matinal « Ton édition vient d'arriver »

> **Type** : Feature · **Cible** : `main` (env staging continu) · **Statut** : CODE+TEST

## Contexte

Facteur a déjà son **moment de fermeture** (closing card « Vous êtes à jour »). Il manque le
**moment d'ouverture** symétrique : au premier open du matin, un écran enveloppe « Bonjour / Ton
édition de {date} vient d'arriver », qui transforme le feed en *édition* datée. C'est le
différenciateur anti-scroll-infini et le cœur de l'identité « facteur ».

L'écran s'affiche **instantanément** (zéro spinner) en lisant l'état déjà préchargé
(`fluxContinuPreloadProvider`, `app.dart:147`). L'utilisateur tape **Ouvrir l'édition**, traverse un
**micro-chargement 2 s** (`LoadingView` existant), puis arrive en **fondu** sur l'Essentiel déjà prêt.

## Décisions verrouillées avec le PO (24/06/2026)

1. **Transition** : réutiliser le `LoadingView` existant (Lottie vélo + citation), re-timé à 2 s —
   zéro nouvel asset.
2. **Fréquence** : 1×/jour, au premier open après la bascule 7h30 Europe/Paris. Les opens suivants
   vont droit au feed.
3. **Numéro d'édition** : abandonné. Date (« mercredi 27 mai ») + « Reçue à 7h00 ».
4. **Édition pas prête** (open à 6h / réseau froid) : ne pas montrer le rituel, filer au feed, et
   **ne pas marquer « vu »** → le rituel revient au prochain open une fois l'édition arrivée.
5. **Méta = sommaire des sections réelles** (et non « 5 sujets · 14 min ») : on tease les sections
   qui apparaîtront vraiment dans l'édition (Technologie · Bon Pote · Actus du jour · Mot du jour ·
   Bonnes Nouvelles), via `FluxContinuState.sections[].label`, libellés **exacts**, **autant que de
   sections réellement affichées**. La Grille = « Mot du jour », insérée à `grilleSlotIndex`.

## Fondations réutilisées (rien à recâbler)

- **Préchargement parallèle** : `fluxContinuPreloadProvider` (`app.dart:147`) lance déjà
  `/digest/both`, `/top-themes`, `/essentiel`, feed + fan-out progressif dès le boot. L'écran rituel
  lit l'état chaud sans jamais lancer ni bloquer de réseau.
- **Gate quotidien 7h30** : `TourneeProgressService.dayKey()` (bascule 7h30 Paris) + clés prefs
  `flux_continu_closing_dismissed_${dayKey}` + `purgeOldPrefsKeys()`. On greffe une clé jumelle.
  Zéro migration, zéro backend.
- **Loader + citations** : `LoadingView(revealEditorialAfter:)` + `EditorialLoaderCard`.
- **Header** : `FacteurLogo` / `StreakIndicator` / `ProfileAvatarButton` (mêmes widgets que
  `_SharedTopHeader`).

## Flux UX cible

1. Boot → splash (auth/onboarding) → redirection.
2. Premier open du jour + pas encore vu → route `/edition` (`MorningRitualScreen`).
   - Affiche immédiatement « Bonjour. » + date FR (local, zéro réseau). Pas de spinner.
   - Dès `editionReady`, révèle en fondu le **sommaire des sections** + le CTA **Ouvrir l'édition →**.
   - Si pas prête après un délai borné (~4 s) → `go('/flux-continu')` **sans marquer vu** (décision #4).
3. Tap CTA → marque « vu aujourd'hui » → haptique légère → cross-fade vers `LoadingView` (2 s) →
   `go('/flux-continu')` en fondu.
4. Essentiel s'affiche instantanément (préchargé) ; fan-out progressif comme aujourd'hui.

### 2. Route + écran
- `config/routes.dart` : `RouteNames.edition='edition'` / `RoutePaths.edition='/edition'`.
  - `postAuthHomePath()` intercepte **avant** flux/flâner :
    `if (!tournee.isMorningRitualShownTodaySync()) return RoutePaths.edition;`
    (lecture **synchrone**, pas d'await, pas de flicker — le gate « prête » est résolu dans l'écran).
  - `GoRoute` `/edition` **top-level** (sibling de splash), `CustomTransitionPage` en fondu.
  - Idempotence : un user sur `/edition` non re-redirigé ; après tap « vu » → `go('/flux-continu')`
    renvoie `null` (pas de rebond).
- `features/flux_continu/screens/morning_ritual_screen.dart` (`ConsumerStatefulWidget`) :
  - Header léger (logo/streak/avatar). Corps : « Bonjour. » (Fraunces) + « Ton édition du {date}
    vient d'arriver. » + fond vélo estompé + « L'Essentiel du jour » + sommaire + CTA.
  - Lecture seule sur `ref.watch(fluxContinuProvider)` / `digestProvider` en `.valueOrNull`.
  - `editionReady = state != null && !state.isSkeleton && state.sections.isNotEmpty && digest != null
    && !digest.isStaleFallback && dayKey(digest.targetDate) == dayKey(now) && digestHasContent`.
  - Timer borné (`_maxWait ≈ 4 s`) : révèle sommaire+CTA dès `editionReady`, sinon
    `go('/flux-continu')` sans marquer vu.
  - Tap CTA : `setMorningRitualShownToday()` → `HapticFeedback.lightImpact()` → cross-fade vers
    `LoadingView(revealEditorialAfter: 300ms)` → après 2 s → `context.go(RoutePaths.fluxContinu)`.

## Hors périmètre
- Pas d'animation d'enveloppe sur-mesure (réutilise le loader). Pas de numéro d'édition.
- Pas de changement backend. Pas de migration Alembic.

## Edge cases
- Open avant 7h / réseau froid : pas prête → feed sans marquer vu. Pendant l'attente : « Bonjour +
  date » seul, jamais de fausse promesse.
- Cache SWR d'hier : `dayKey(targetDate) == today && !isStaleFallback` empêche un rituel périmé.
- Background → foreground même jour : redirection inchangée ; « vu » déjà marqué → pas de re-montée.
- Reduce motion : fondu simple.

## Tests
- `TourneeProgressService` : `isMorningRitualShownTodaySync` faux par défaut, vrai après set, purge
  des vieilles clés, bascule au `dayKey` 7h30.
- Helpers : date FR déterministe ; `editionSummaryEntries` (héros exclu, ordre feed, « Mot du jour »
  au bon index, suggestions incluses, aucun libellé inventé).
- Widget `MorningRitualScreen` : greeting instantané sans provider prêt (pas de spinner) ; révélation
  sommaire+CTA à `editionReady` ; forward-to-feed sans marquer vu si pas prête.
