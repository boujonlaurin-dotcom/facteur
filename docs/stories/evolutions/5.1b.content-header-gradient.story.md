# Story 5.1b: Gradient immersif header article

**Parent Story:** [[5.1.content-detail-screen.story.md]]  
**Epic:** 5 - Consommation & Gamification  
**Classification:** EVOLUTION

---

## Description

**En tant qu'** utilisateur lisant un article,  
**Je veux** que le bas du header ait un dégradé transparent,  
**afin de** rendre la transition vers le contenu plus fluide et immersive.

---

## Contexte

Le header actuel du `ContentDetailScreen` a un gradient qui va de `backgroundPrimary` 100% → transparent, mais uniquement sur le haut. Cette story ajoute un effet similaire mais inversé sur le bas du header pour créer une meilleure transition vers la zone de contenu (WebView/Reader).

---

## Critères d'acceptation

1. ✅ Le bas du header (container avec Source, Date, Comparer) a un dégradé vertical
2. ✅ Direction du dégradé: `Alignment.topCenter` → `Alignment.bottomCenter`
3. ✅ Couleurs: `backgroundPrimary` (100%) → `backgroundPrimary.withOpacity(0.85)` 
4. ✅ L'effet est visible en mode clair ET sombre
5. ✅ Pas d'impact sur les performances de scroll

---

## Implémentation technique

### Fichier modifié
- `apps/mobile/lib/features/detail/screens/content_detail_screen.dart`

### Changement
Dans `_buildHeader()`, modifier le `BoxDecoration.gradient`:

```dart
decoration: BoxDecoration(
  gradient: LinearGradient(
    begin: Alignment.topCenter,
    end: Alignment.bottomCenter,
    colors: [
      colors.backgroundPrimary,
      colors.backgroundPrimary.withOpacity(0.85), // Fin semi-transparent
    ],
  ),
),
```
