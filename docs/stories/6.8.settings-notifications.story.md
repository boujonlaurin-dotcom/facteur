# Story 6.8 : Écran Paramètres Notifications

**As a** utilisateur,
**I want** gérer mes préférences de notifications,
**so that** je contrôle les alertes que je reçois de l'application.

## Context
Aucun système de notification n'existe actuellement. Cette story pose les bases avec des préférences locales.

## Acceptance Criteria

1. Un tile "Notifications" dans Settings mène à un nouvel écran.
2. L'écran affiche des toggles pour :
   - **Notifications Push** (ON par défaut pour MVP, mais pas encore connecté à un service push).
   - **Emails périodiques** (Résumé hebdo, newsletter – OFF par défaut).
3. Les préférences sont persistées localement (Hive `settings` box).
4. Les changements sont appliqués instantanément (pas de bouton "Sauvegarder").

## Out of Scope (V0)
- Intégration réelle avec un service de Push Notifications (Firebase, OneSignal).
- Intégration avec un service d'envoi d'emails (SendGrid, Mailchimp).

## Technical Notes
- Nouveau provider Riverpod `notificationsSettingsProvider` pour lire/écrire Hive.
- Structure Hive : `{ push_enabled: bool, email_digest_enabled: bool }`.
