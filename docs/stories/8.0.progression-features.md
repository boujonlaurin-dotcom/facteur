# Story 8: Système de Progression & Suivi de Thèmes

**Status:** Implemented (Repair Phase)

## But
Permettre à l'utilisateur de "suivre" des thèmes spécifiques (ex: Tech, Politique) directement depuis sa consommation de contenu, pour ensuite débloquer des quiz et mesurer sa progression.

## Fonctionnalités Clés

### 1. Suivi de Thème (Follow Topic)
-   **Mécanisme** : Le suivi remplace l'ancien système de "Bookmark" pour la gamification.
-   **Entrée Utilisateur** :
    -   **Dynamic Feed Card** : Après avoir lu un article pendant plus de 30 secondes, une carte d'invitation apparaît dynamiquement sous l'article dans le flux.
    -   **Message** : "Vous avez lu sur [Thème] ? Ne perdez pas le fil."
    -   **Action** : "Suivre ce sujet".

### 2. Expérience de Lecture (Article Reader)
-   **Contenu In-App** : Uniquement affiché si le HTML complet est disponible.
-   **Fallback** : Si le contenu est partiel (description uniquement), l'article s'ouvre dans une **WebView** (site original) pour garantir une expérience de lecture qualitative.
-   **UI** : Pas de footer intrusif. La lecture est épurée.

### 3. Gamification (Quiz)
-   Une fois un thème suivi, l'utilisateur a accès aux Quiz associés (via l'onglet Progression ou invitation directe).
-   *Note: Le lien vers le Quiz est proposé directement sur la carte de progression si le thème est déjà suivi.*

## Implementation Technique
-   **Frontend** : `FeedScreen` gère une map locale `_activeProgressions` pour insérer les cartes à la volée.
-   **Widget** : `ProgressionCard` (Design "Feed Native").
-   **Backend** : Endpoints existants pour `follow_topic` et `get_progression`.
