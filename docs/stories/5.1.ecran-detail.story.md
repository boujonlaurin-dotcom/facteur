# Story 5.1 : Écran Détail Contenu

**As a** utilisateur,
**I want** voir un aperçu enrichi avant d'ouvrir un contenu,
**so that** je puisse décider si je veux vraiment le consulter.

## Contexte
Actuellement, le tap sur une carte ouvre directement l'URL. Nous voulons introduire un écran intermédiaire de "Détail" qui offre plus de contexte (description, métadonnées complètes) et permet des actions (Sauvegarder, Masquer) avant l'engagement de lecture complète.

## Maquettes / Design
- **Header** : Image (Thumbnail) en grand format (ratio 16:9 ou 4:3).
- **Body** :
    - Titre (H1).
    - Source (Logo + Nom) & Métadonnées (Date, Durée).
    - Description/Résumé (si disponible dans le flux RSS).
- **Footer / Actions** :
    - **CTA Principal** : "Lire l'article" / "Voir la vidéo" / "Écouter" (Ouvre l'URL/WebView).
    - **Actions Secondaires** :
        - Bouton Sauvegarder (Bookmark)
        - Bouton "Pas intéressé" (Menu "..." ou bouton dédié)

## Acceptance Criteria
- [ ] **Navigation** : Le tap sur une `FeedCard` navigue vers `ContentDetailScreen` (au lieu d'ouvrir l'URL directement).
- [ ] **UI Détail** : L'écran affiche toutes les infos du `Content`.
- [ ] **Action Lecture** : Le CTA principal déclenche l'ouverture de l'URL (WebView/Browser) et le tracking "Consommé".
- [ ] **Action Sauvegarde** : Bouton pour toggler l'état sauvegardé.
- [ ] **Action Masquer** : Bouton pour masquer le contenu (retour au feed).
- [ ] **Transition** : Hero animation sur l'image si possible pour une transition fluide depuis le feed.
