# Story 4.8 — Feed Carousel Engagement (Position & Variability)

## Contexte

Les carousels du feed souffrent de deux problèmes d'engagement :
1. Le premier carousel visible apparaît trop tôt (position 1-3), interrompant le flux de lecture avant que l'utilisateur ait scanné assez d'articles.
2. Les carousels sont trop prévisibles : même position, même ordre, même contenu sélectionné chaque jour.

## Objectif

Améliorer l'engagement des carousels via deux leviers :
- **B — Plancher + rotation** : aucun carousel avant la position 4, et shuffle quotidien des slots entre carousels.
- **C — Contenu probabiliste** : sélection pondérée parmi les top-N candidats au lieu du strict #1.

## Acceptance Criteria

- [ ] AC1: Aucun carousel n'a une position < 4 (MIN_CAROUSEL_POSITION = 4)
- [ ] AC2: Les positions des carousels varient d'un jour à l'autre pour un même utilisateur
- [ ] AC3: Un même user + même date produit toujours les mêmes positions (déterministe intra-jour)
- [ ] AC4: Le hot cluster est sélectionné parmi les top-3 clusters éligibles (pondéré par taille)
- [ ] AC5: L'ordre interne du carousel gems est shufflé quotidiennement
- [ ] AC6: Pas de changement frontend requis
- [ ] AC7: Tous les tests existants passent (adaptés si nécessaire)

## Plan technique

### B1. Plancher minimum (MIN_CAROUSEL_POSITION = 4)
- Fichier : `packages/api/app/services/recommendation_service.py`
- Constante `MIN_CAROUSEL_POSITION = 4` dans `_build_carousels()`
- Appliqué via `max(position, MIN_CAROUSEL_POSITION)` à chaque carousel

### B2. Shuffle quotidien des slots
- 3 slots : [4-6], [8-11], [14-17]
- Seed = `hash(str(user_id) + str(date.today()))` → déterministe par jour/user
- Les carousels éligibles sont assignés aux slots dans un ordre shufflé
- Fallback : sans user_id, positions par défaut (pas de shuffle)

### C1. Hot cluster probabiliste
- Fichier : `packages/api/app/services/article_clustering_service.py`
- `find_hot_cluster()` : prendre top-3 clusters >= min_items, tirer un cluster pondéré par taille
- Nouveau paramètre optionnel `seed: int | None` pour le déterminisme

### C2. Gems shuffle interne
- Fichier : `packages/api/app/services/recommendation_service.py`
- Après le tri par score, shuffle Fisher-Yates avec le même seed quotidien

### Tests
- Fichier : `packages/api/tests/test_feed_carousels.py`
- Adapter tests de position (ranges au lieu de valeurs exactes)
- Nouveaux tests : plancher, déterminisme, variabilité inter-jours
