## Story

**As a** lecteur quotidien de Facteur,
**I want** un digest compact et scannable avec une hiérarchie claire, des indicateurs de couverture médiatique, et la possibilité d'approfondir chaque sujet en un tap,
**so that** je peux parcourir l'essentiel de l'actualité en 2 minutes, comprendre quels sujets sont les plus couverts/controversés, et accéder facilement à la comparaison de sources — le différenciant Facteur.

## Contexte & Motivation

Le digest `editorial_v1` actuel (3 sujets) ressemble à un feed :
- **Pas de hiérarchie** : toutes les cartes ont le même poids visuel (~420px chacune)
- **Intros génériques** : les textes LLM ne donnent pas de "quick view" unique
- **Comparaison de sources cachée** : le CTA "Comparer" n'est accessible que depuis le reader, pas depuis le digest
- **Pas assez d'actus** : 3 sujets ne suffisent pas à créer le sentiment de "finitude" (l'utilisateur sent qu'il manque des choses)

Cette refonte introduit :
1. **5+2 articles** (5 sujets actus + pépite + coup de cœur)
2. **Toggle compact/étendu** sur chaque carte
3. **Badge "Couvert par X sources"** avec données du PerspectiveService
4. **Analyse rapide des divergences médiatiques** dans l'état étendu
5. **CTA "Comparer les sources"** directement dans le digest
6. **Traitement "À la Une"** pour le sujet #1

## Acceptance Criteria

### Backend

1. Le pipeline éditorial sélectionne **5 sujets** (au lieu de 3), avec 1 "À la Une" + 4 sujets complémentaires
2. Pour chaque sujet, le PerspectiveService est appelé **en batch à la génération** pour obtenir :
   - `perspective_count` : nombre réel de sources couvrant le sujet (recherche hybride DB + Google News RSS)
   - `bias_distribution` : dict `{"left": N, "center-left": N, "center": N, "center-right": N, "right": N}`
   - `bias_highlights` : string descriptif ("Très couvert à gauche", "Aucun média de droite", "Couverture équilibrée")
   - `divergence_analysis` : texte LLM ~150 mots analysant les angles médiatiques (si ≥3 perspectives)
3. Les 4 champs sont stockés dans le JSON du digest et exposés dans la réponse API `DigestTopic`
4. Les anciens digests (sans ces champs) restent lisibles — tous les nouveaux champs ont des defaults (`0`, `None`)
5. Le prompt d'écriture (`writing` + `writing_serene`) génère des **intros factuelles denses** (~40 mots max) qui NE mentionnent PAS le nombre de sources ni les divergences (c'est l'UI qui s'en charge)
6. Le deep matcher utilise des seuils élargis pour le sujet "À la Une" (`prefilter_limit` ×2, `threshold` /2)
7. Les appels PerspectiveService sont parallélisés (`asyncio.gather`) et ont un fallback : si timeout/erreur → `perspective_count` fallback sur `source_count` du cluster, `divergence_analysis` = None

### Frontend

8. Chaque topic dans le digest éditorial a un **mécanisme de toggle** expand/collapse
9. **État fermé (compact, par défaut)** : image réduite + titre + badge "Couvert par X sources" (avec icône trending ↑) + intro 1 ligne tronquée + source + icône expand
10. **État ouvert (étendu)** : image pleine 16:9 + titre + badge sources + intro complète + bloc analyse divergences + bloc "Pas de recul" (si dispo) + CTA "Comparer les sources" + CTA "Lire"
11. Le topic avec `isUne == true` a un badge **"À la Une"** visible en état fermé
12. Le CTA "Comparer les sources" ouvre la **bottom sheet perspectives existante** (`perspectives_bottom_sheet.dart`)
13. Le bloc "Pas de recul" s'affiche uniquement si un article avec `badge == "pas_de_recul"` existe dans `topic.articles`
14. Le carousel horizontal (multi-articles par topic) est **conservé**
15. Les formats `topics_v1` et `flat_v1` ne sont **pas impactés** (0 régression)

### ÉTAPE 3C — PerspectiveService batch (détail)

```python
async def _compute_perspectives_batch(
    self,
    subjects: list[EditorialSubject],
    cluster_map: dict[str, TopicCluster],
) -> None:
    """Enrichit chaque sujet avec les données de perspectives (parallèle)."""

    perspective_service = PerspectiveService(db=self.db)

    async def process_one(subject: EditorialSubject) -> None:
        cluster = cluster_map.get(subject.topic_id)
        if not cluster or not cluster.contents:
            subject.perspective_count = 0
            return

        # Article représentatif = le plus récent du cluster
        representative = max(cluster.contents, key=lambda c: c.published_at)

        try:
            perspectives, _ = await perspective_service.get_perspectives_hybrid(
                content=representative,
                exclude_domain=getattr(representative, 'source_url_domain', None),
            )
        except Exception:
            # Fallback sur le source_count du cluster
            subject.perspective_count = len(cluster.source_ids)
            return

        subject.perspective_count = len(perspectives)
        subject.bias_distribution = _compute_bias_distribution(perspectives)
        subject.bias_highlights = _compute_bias_highlights(subject.bias_distribution)

        if len(perspectives) >= 3:
            try:
                subject.divergence_analysis = await perspective_service.analyze_divergences(
                    article_title=representative.title,
                    source_name=getattr(representative, 'source_name', ''),
                    source_bias=perspective_service.resolve_bias(representative),
                    perspectives=[_to_perspective_dict(p) for p in perspectives],
                    article_description=getattr(representative, 'description', None),
                )
            except Exception:
                subject.divergence_analysis = None

    await asyncio.gather(
        *(process_one(s) for s in subjects),
        return_exceptions=True,
    )
```

### Helper `_compute_bias_highlights()`

```python
def _compute_bias_highlights(bias_dist: dict[str, int] | None) -> str | None:
    if not bias_dist:
        return None

    total = sum(bias_dist.values())
    if total == 0:
        return None

    left = bias_dist.get("left", 0) + bias_dist.get("center-left", 0)
    right = bias_dist.get("right", 0) + bias_dist.get("center-right", 0)
    center = bias_dist.get("center", 0)

    # Absence totale d'un bord
    if right == 0 and left >= 2:
        return "Aucun média de droite"
    if left == 0 and right >= 2:
        return "Aucun média de gauche"

    # Dominance forte (>60%)
    if left / total > 0.6:
        return "Très couvert à gauche"
    if right / total > 0.6:
        return "Très couvert à droite"

    return "Couverture équilibrée"
```

### Données PerspectiveService disponibles

Le `PerspectiveService.get_perspectives_hybrid()` retourne :
- `list[Content]` — articles couvrant le même sujet (DB + Google News RSS)
- Chaque Content a : `title`, `source_name`, `description`, `published_at`

Le `PerspectiveService.analyze_divergences()` retourne :
- `str | None` — texte ~150 mots (Mistral Large, temperature 0.4, max_tokens 300)
- Format : 1-2 phrases de contexte + constats "→" nommant les médias

Le `resolve_bias()` utilise un cascade 4 niveaux :
1. `DOMAIN_BIAS_MAP` hardcodé (45 sources françaises)
2. Cache in-memory
3. DB lookup (source domain/name fuzzy match)
4. Default: "unknown"

## Risques & Mitigations

| Risque | Probabilité | Impact | Mitigation |
|--------|-------------|--------|------------|
